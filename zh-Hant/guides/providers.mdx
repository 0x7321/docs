---
title: "供應商與路由"
description: "同一模型由多個供應商承載，如何在令牌中配置或按請求覆蓋路由。"
---

每個模型可能由**多個供應商**承載。供應商是按來源和成本組織的服務節點，不同供應商有不同的價格倍率和模型覆蓋。傳送請求時，系統按你的令牌配置自動選擇可用供應商；某個供應商失敗或超時會自動重試另一個，對你的程式碼透明。

## 可用供應商

| ID | 名稱 | 說明 |
| --- | --- | --- |
| `official` | 官方 | 聚合 OpenAI、Anthropic、Google 等各大官方直連的供應商組，穩定性最高 |
| `official_cheap` | 官方（折扣） | 官方渠道的成本最佳化版，穩定性高、價格更低 |
| `mix` | 混合 | 多源混合渠道，價格最低、模型覆蓋最廣 |

`official` 是一個**供應商組**，打包了各大官方平臺的節點；選它時系統在所有官方節點間智慧路由與故障轉移，無需逐家指定。

### 怎麼選

| 你的需求 | 推薦配置 |
| --- | --- |
| 穩定優先，不在意價格 | `official` |
| 穩定且更划算 | `official_cheap` |
| 成本最低，可接受輕微不穩定 | `mix` |
| 先折扣官方，失敗再回退混合 | `official_cheap,mix` |

## 在令牌中配置

建立或編輯令牌時可設定該令牌的預設路由：

* **供應商列表**：留空（推薦）= 在所有可用供應商間自動路由；單個 = 只走該供應商；多個用逗號分隔，按順序或排序策略嘗試。
* **僅限指定供應商**：關（預設）= 優先指定的，全失敗再回退其它；開 = 嚴格只用指定的，全失敗直接報錯不回退。
* **排序策略**：手動（預設，按你列出的順序）/ 延遲 / 價格 / 吞吐。

## 按請求覆蓋

也可用請求頭對**單次請求**臨時覆蓋令牌設定，無需改令牌：

| 請求頭 | 作用 |
| --- | --- |
| `X-Provider-Order` | 指定本次請求的供應商及順序 |
| `X-Provider-Only` | `"true"` 開啟嚴格模式，不回退 |
| `X-Provider-Sort` | 覆蓋排序策略（`latency` / `price` / `throughput`） |
| `X-Provider-Ignore` | 本次請求跳過的供應商，疊加在令牌配置之上 |

> 透過請求頭指定的供應商仍受你的賬戶許可權約束。

### 示例

```bash
# 先折扣官方，失敗回退混合
curl https://bearlab.ai/v1/chat/completions \
  -H "Authorization: Bearer sk-your-token" \
  -H "X-Provider-Order: official_cheap,mix" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o", "messages": [{"role": "user", "content": "Hello"}]}'
```

```bash
# 嚴格只走官方，不回退
curl https://bearlab.ai/v1/chat/completions \
  -H "Authorization: Bearer sk-your-token" \
  -H "X-Provider-Order: official" \
  -H "X-Provider-Only: true" \
  -H "Content-Type: application/json" \
  -d '{"model": "claude-sonnet-4-5", "messages": [{"role": "user", "content": "Hello"}]}'
```

也可在 SDK 客戶端裡設預設頭：

```python
import openai

client = openai.OpenAI(
    api_key="sk-your-token",
    base_url="https://bearlab.ai/v1",
    default_headers={"X-Provider-Order": "official_cheap,mix"},
)
```
