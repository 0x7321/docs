## En-têtes de la requête

| En-tête | Obligatoire | Description |
| --- | --- | --- |
| `Authorization` | ✓ | `Bearer YOUR_ACCESS_TOKEN`, jeton d'accès du compte (état de connexion à la console, pas une clé API `sk-`) |

Interface : `GET /api/token/{id}`

Récupérez la configuration détaillée d'un seul jeton en fonction de son Token ID. La clé `key` dans la réponse est masquée ; pour obtenir la **clé en texte brut complet**, utilisez `POST /api/token/{id}/key`.

## Paramètres de chemin

| Paramètre | Obligatoire | Description |
| --- | --- | --- |
| `id` | ✓ | Token ID, provenant de `data.items[].id` de `GET /api/token/` |

```bash
curl https://bearlab.ai/api/token/10665 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

<Note>
Source : code source new-api `controller/token.go` `GetToken`, la réponse `data` est `buildMaskedTokenResponse(token)` (la `key` est masquée). L'obtention de la clé en texte brut complet est une interface distincte `POST /api/token/{id}/key` (`GetTokenKey`).
</Note>
