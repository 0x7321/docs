Chaque modèle sur BearLab AI est servi par un ou plusieurs **fournisseurs en amont** (vendeurs). Derrière un seul nom de modèle, la passerelle achemine votre demande vers un nœud fournisseur sain ; en cas d'échec ou de délai d'attente, elle réessaie un autre de manière transparente, sans que votre code s'en aperçoive. Les fournisseurs sont organisés en **groupes de routage**, et votre jeton détermine de quel(s) groupe(s) il peut puiser.

<Note>
BearLab AI est une passerelle unifiée construite sur New API. Le catalogue des fournisseurs ci-dessous est généré à partir des données de prix en direct pour ce compte, il reflète donc exactement quels vendeurs et familles de modèles sont accessibles avec votre clé.
</Note>

## URLs de base

Choisissez la région la plus proche de vous. Les deux régions exposent les mêmes modèles et le même routage des fournisseurs.

| Région | URL de base compatible OpenAI | URL de base Anthropic |
| --- | --- | --- |
| Amériques / UE | `https://bearlab.ai/v1` | `https://bearlab.ai` (messages à `/v1/messages`) |
| Asie-Pacifique | `https://bearlab.space/v1` | `https://bearlab.space` (messages à `/v1/messages`) |

L'authentification est identique partout : envoyez votre clé API en tant que jeton Bearer.

```
Authorization: Bearer sk-...
```

Chaque fournisseur correspond à une route de vendeur en amont. Le tableau est construit à partir du mappage en direct des modèles aux vendeurs ; les chiffres représentent le nombre de variantes de modèles actuellement tarifées sous ce fournisseur.

| Fournisseur | Vendeur en amont | Familles de modèles | Variantes | Protocoles |

| Google | Google Gemini | Gemini 2.5 / 3 / 3.1 / 3.5 (variantes Pro, Flash, image, CLI) | 79 | OpenAI + Gemini |
| OpenAI | OpenAI | GPT-5.x, GPT-image-2 | 23 | OpenAI |
| xAI | xAI | Grok 4 / 4.1 / 4.2 / 4.3 | 10 | OpenAI |
| Alibaba Bailian (agrégateur) | Alibaba Cloud Bailian | Qwen, GLM, DeepSeek, Kimi, MiniMax | 18 | OpenAI + Anthropic |

<Note>
Certaines familles de modèles sont accessibles par plus d'un fournisseur. Par exemple, les modèles nationaux (Qwen / GLM / DeepSeek / Kimi / MiniMax) sont tous desservis par l'agrégateur **Alibaba Bailian** plutôt que par des routes de vendeurs natifs distincts. La passerelle choisit une route disponible pour vous, vous n'avez donc rarement besoin de nommer un fournisseur spécifique.
</Note>

<Warning>
L'accès aux groupes est lié à votre clé. Nommer un groupe ou un fournisseur pour lequel vous n'avez pas la permission dans une demande ne le déverrouillera pas — la passerelle n'achemine que dans ce que votre jeton est autorisé à faire.
</Warning>

## Configuration du routage dans un jeton

Lorsque vous créez ou modifiez un jeton, vous définissez son routage par défaut :

- **Liste des fournisseurs / groupes** : laisser vide (recommandé) pour acheminer à travers tout ce que votre clé peut atteindre ; définir une seule entrée pour restreindre à celle-ci ; définir plusieurs (séparées par des virgules) pour les essayer dans l'ordre ou selon la stratégie de tri.
- **Mode fournisseur uniquement** : désactivé (par défaut) donne la priorité aux fournisseurs listés mais revient à d'autres ; activé limite strictement à ceux-ci et renvoie une erreur si tous échouent.
- **Stratégie de tri** : manuel (par défaut), latence, prix ou débit.

## Remplacement par demande

Remplacez les défauts de votre jeton pour une seule demande via des en-têtes :

| En-tête | Effet |
| --- | --- |
| `X-Provider-Order` | Fournisseurs / groupes et ordre à utiliser pour cette demande |
| `X-Provider-Only` | `"true"` active le mode strict (pas de secours) |
| `X-Provider-Sort` | Remplacer la stratégie de tri (`latency` / `price` / `throughput`) |
| `X-Provider-Ignore` | Fournisseurs à ignorer, empilés sur la configuration du jeton |

> Les fournisseurs et groupes demandés via les en-têtes sont toujours soumis à vos permissions de compte.

### Exemple

<CodeGroup>

```bash Amériques / UE (bearlab.ai)
curl https://bearlab.ai/v1/chat/completions \
  -H "Authorization: Bearer sk-..." \
  -H "Content-Type: application/json" \
  -H "X-Provider-Sort: latency" \
  -d '{
    "model": "gpt-5.5",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

```bash Asie-Pacifique (bearlab.space)
curl https://bearlab.space/v1/chat/completions \
  -H "Authorization: Bearer sk-..." \
  -H "Content-Type: application/json" \
  -H "X-Provider-Sort: latency" \
  -d '{
    "model": "gpt-5.5",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

</CodeGroup>

## Découverte des fournisseurs en direct

La liste ci-dessus est un instantané. Pour voir exactement quels modèles votre clé peut atteindre maintenant, interrogez le point de terminaison des modèles — la passerelle détecte automatiquement le protocole à partir de vos en-têtes d'authentification :

<CodeGroup>

```bash Protocole OpenAI
curl https://bearlab.ai/v1/models \
  -H "Authorization: Bearer sk-..."
```

```bash Protocole Anthropic
curl https://bearlab.ai/v1/models \
  -H "x-api-key: sk-..." \
  -H "anthropic-version: 2023-06-01"
```

</CodeGroup>

Chaque entrée inclut `id`, `owned_by` et `supported_endpoint_types`, vous pouvez donc dire quels protocoles un modèle donné accepte avant de l'appeler. Voir [Models](/fr/guides/models) pour le catalogue complet et [Pricing](/fr/guides/pricing) pour les tarifs par modèle.
