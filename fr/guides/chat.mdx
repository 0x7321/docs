La complétion de chat est le point de terminaison principal, offert via trois interfaces compatibles (OpenAI, Anthropic, Gemini) partageant une clé unique.

<Note>
  BearLab AI dessert deux régions. Les utilisateurs américains/européens appellent `https://bearlab.ai` ; les utilisateurs d'Asie-Pacifique appellent le domaine dédié `https://bearlab.space`. Les deux régions partagent la même clé et le même catalogue de modèles. Voir [Disponibilité du Service](/fr/guides/availability).
</Note>

## Points de terminaison

| Protocole | Méthode et chemin (US / EU) | Méthode et chemin (Asie-Pacifique) |
| --- | --- | --- |
| Compatible OpenAI | `POST https://bearlab.ai/v1/chat/completions` | `POST https://bearlab.space/v1/chat/completions` |
| Compatible Anthropic | `POST https://bearlab.ai/v1/messages` | `POST https://bearlab.space/v1/messages` |
| Gemini natif | `base_url=https://bearlab.ai`, `api_version=v1beta` (SDK Google genai) | `base_url=https://bearlab.space`, `api_version=v1beta` |

Authentifiez chaque requête avec votre clé API (`sk-...`) dans l'en-tête `Authorization: Bearer`. Le protocole Anthropic accepte également l'en-tête `x-api-key`.

## Paramètres courants

| Paramètre | Requis | Description |
| --- | --- | --- |
| `model` | Oui | ID du modèle. Voir [API des Modèles et Tarification](/fr/guides/models). |
| `messages` | Oui | Tableau ordonné de messages de conversation. |
| `max_tokens` | OpenAI : optionnel · Anthropic : **requis** | Nombre maximum de tokens de sortie. |
| `temperature` | Non | Température d'échantillonnage. Plage OpenAI `0`-`2` ; plage Anthropic `0`-`1`. |
| `stream` | Non | Diffusez la réponse via SSE lorsqu'il est défini sur `true`. |
| `tools` | Non | Définitions de fonctions / outils pour les appels d'outils. |

<Warning>
  Différence de protocole : le protocole OpenAI place l'invite système dans `messages` comme `role: system`, et `max_tokens` est optionnel. Le protocole Anthropic utilise un champ `system` de haut niveau et **exige** `max_tokens`.
</Warning>

## Streaming

Définissez `stream: true` pour recevoir la réponse via Server-Sent Events.

- **Protocole OpenAI** — portions `delta` incrémentielles, terminées par une ligne finale `data: [DONE]`.
- **Protocole Anthropic** — événements SSE typés (`message_start`, `content_block_delta`, `message_stop`, …) ; il n'y a pas de sentinelle `[DONE]`.

<CodeGroup>
```python US / EU
from openai import OpenAI

client = OpenAI(api_key="sk-...", base_url="https://bearlab.ai/v1")

stream = client.chat.completions.create(
    model="gpt-5.5",
    messages=[{"role": "user", "content": "Write a short poem about spring."}],
    stream=True,
)
for chunk in stream:
    delta = chunk.choices[0].delta.content
    if delta:
        print(delta, end="", flush=True)
```

```python Asie-Pacifique
from openai import OpenAI

client = OpenAI(api_key="sk-...", base_url="https://bearlab.space/v1")

stream = client.chat.completions.create(
    model="gpt-5.5",

    stream=True,
)

```
</CodeGroup>

## Appels d'outils

Les protocoles OpenAI et Anthropic prennent tous deux en charge les appels d'outils.

```python
from openai import OpenAI

client = OpenAI(base_url="https://bearlab.ai/v1", api_key="sk-...")

tools = [{
    "type": "function",
    "function": {
        "name": "get_weather",
        "description": "Get the current weather for a city",
        "parameters": {
            "type": "object",
            "properties": {"city": {"type": "string"}},
            "required": ["city"],
        },
    },
}]

resp = client.chat.completions.create(
    model="gpt-5.5",
    messages=[{"role": "user", "content": "What's the weather in London?"}],
    tools=tools,
)
print(resp.choices[0].message)
```

<Note>
  Utilisateurs d'Asie-Pacifique : définissez `base_url="https://bearlab.space/v1"` à la place.
</Note>

## Protocole natif Gemini

Les modèles Gemini peuvent être appelés au format natif Google AI Studio, sans conversion au format OpenAI.

<CodeGroup>
```python US / EU
from google import genai
from google.genai import types

client = genai.Client(
    api_key="sk-...",
    http_options=types.HttpOptions(api_version="v1beta", base_url="https://bearlab.ai"),
)

resp = client.models.generate_content(model="gemini-3.1-pro-preview", contents="Hello")
print(resp.text)
```

```python Asie-Pacifique
from google import genai
from google.genai import types

client = genai.Client(
    api_key="sk-...",
    http_options=types.HttpOptions(api_version="v1beta", base_url="https://bearlab.space"),
)

resp = client.models.generate_content(model="gemini-3.1-pro-preview", contents="Hello")
print(resp.text)
```
</CodeGroup>

<Warning>
  1. Définissez `base_url` sur la racine de la région (`https://bearlab.ai` ou `https://bearlab.space`) ; n'ajoutez pas `/v1beta`, le SDK l'ajoute.
  2. Définissez `api_version` sur `v1beta`.
  3. Installer : Python `pip install google-genai`, Node.js `npm install @google/genai`.
</Warning>

<Tip>
  Les préfixes longs répétés (invites système fixes, longs documents) peuvent utiliser [Mise en cache des invites](/fr/guides/prompt-caching) pour réduire le coût de la partie répétée.
</Tip>
