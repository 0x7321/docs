Cette page liste les modèles que votre clé API peut appeler, regroupés par famille. Passez le `id` exact d'un modèle dans le champ `model` d'une demande de chat.

<Note>
  Utilisateurs d'Asie-Pacifique : utilisez le domaine dédié `https://bearlab.space` au lieu de `https://bearlab.ai`. Voir [Disponibilité du service](/fr/guides/availability).
</Note>

## Comment lire ce catalogue

Le catalogue que vous voyez est lié à **votre groupe de jetons**. Chaque clé API appartient à un groupe de modèles, et un modèle n'apparaît dans votre liste que si votre groupe est autorisé à l'appeler. Les modèles ci-dessous constituent la gamme de production actuelle ; le sous-ensemble exact visible pour votre clé dépend de votre groupe.

| Concept | Signification |
| --- | --- |
| **Modèle `id`** | La chaîne exacte que vous envoyez dans `model`. Sensible à la casse, copiez-la littéralement. |
| **Groupe de modèles** | Le niveau d'accès auquel appartient votre clé. Détermine les modèles que vous pouvez appeler et à quel prix. |
| **Type d'endpoint** | Le protocole qu'un modèle accepte : compatible OpenAI, Anthropic ou Gemini. |

## Récupérer la liste en direct

`/v1/models` retourne les modèles que votre clé peut appeler, au format OpenAI SDK. Il nécessite votre clé API. Le format de réponse suit le format impliqué par vos en-têtes de demande (OpenAI par défaut ; Anthropic avec `x-api-key` + `anthropic-version` ; Gemini avec `x-goog-api-key`).

<Tabs>
  <Tab title="États-Unis / UE (bearlab.ai)">
    ```bash
    curl https://bearlab.ai/v1/models \
      -H "Authorization: Bearer sk-..."
    ```
  </Tab>
  <Tab title="Asie-Pacifique (bearlab.space)">
    ```bash
    curl https://bearlab.space/v1/models \
      -H "Authorization: Bearer sk-..."
    ```
  </Tab>
</Tabs>

Réponse (tronquée) :

```json
{
  "object": "list",
  "data": [
    {
      "id": "claude-opus-4-8",
      "object": "model",
      "created": 1626777600,
      "owned_by": "openai",
      "supported_endpoint_types": ["openai"]
    }
  ]
}
```

| Champ | Description |
| --- | --- |
| `id` | Identifiant du modèle ; passez ceci dans le champ `model` d'une demande de chat |
| `object` | Toujours `model` |
| `created` | Temps de sortie (secondes Unix) |
| `owned_by` | Chaîne du propriétaire nominal |
| `supported_endpoint_types` | Quels endpoints acceptent ce modèle : `openai`, `anthropic`, et/ou `gemini` |

## Appeler un modèle

Envoyez le `id` exact dans le champ `model`. Les deux domaines d'exemple sont affichés ; choisissez celui de votre région.

<Tabs>
  <Tab title="États-Unis / UE (bearlab.ai)">
    ```bash
    curl https://bearlab.ai/v1/chat/completions \
      -H "Authorization: Bearer sk-..." \
      -H "Content-Type: application/json" \
      -d '{
        "model": "claude-sonnet-4-6",
        "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
      }'
    ```
  </Tab>
  <Tab title="Asie-Pacifique (bearlab.space)">
    ```bash
    curl https://bearlab.space/v1/chat/completions \
      -H "Authorization: Bearer sk-..." \
      -H "Content-Type: application/json" \
      -d '{
        "model": "claude-sonnet-4-6",
        "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
      }'
    ```
  </Tab>
</Tabs>
