---

[Codex CLI](https://github.com/openai/codex) est l'agent de codage officiel en ligne de commande d'OpenAI, connecté via un fournisseur personnalisé.

<Warning>
  Codex utilise l'**API Responses** d'OpenAI (`wire_api = "responses"`), un protocole propriétaire OpenAI qui **fonctionne uniquement avec les modèles OpenAI** (`gpt-*` / `o*`). Les modèles Claude, Gemini et autres ne peuvent pas être utilisés via Codex — pour Claude, utilisez [Claude Code](/fr/integrations/claude-code), [Cursor](/fr/integrations/cursor) ou [Cline](/fr/integrations/cline).
</Warning>

## Modèles recommandés

| Modèle | ID | Idéal pour |
| --- | --- | --- |
| GPT-5.5 | `gpt-5.5` | Dernier modèle généraliste, défaut actuel de Codex |
| GPT-5.2-Codex | `gpt-5.2-codex` | Optimisé Codex pour le codage, idéal en workflow clé API |
| GPT-5.4-mini | `gpt-5.4-mini` | Léger et rapide, sous-tâches et éditions interactives |

## Configuration

Modifiez `~/.codex/config.toml` et ajoutez un fournisseur pointant vers BearLab :

```toml
model = "gpt-5.5"
model_provider = "bearlab"

# Les fournisseurs tiers doivent désactiver le stockage des réponses
disable_response_storage = true

[model_providers.bearlab]
name = "BearLab AI"
base_url = "https://bearlab.ai/v1"
env_key = "BEARLAB_API_KEY"
wire_api = "responses"
```

Exportez la clé et exécutez :

```bash
export BEARLAB_API_KEY="YOUR_API_KEY"
codex
```

<Note>
  Utilisez l'adresse compatible OpenAI se terminant par `/v1`. `wire_api = "responses"` et `disable_response_storage = true` sont requis pour un fournisseur tiers au format OpenAI. `model` doit être un ID de la famille OpenAI existant dans le [point de terminaison des modèles](/fr/guides/models).
</Note>
