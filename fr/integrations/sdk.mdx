---
title: "SDK officiels (Python / Node)"
description: "Utilisez les SDK officiels OpenAI ou Anthropic avec BearLab AI."
---

Les SDK officiels OpenAI et Anthropic fonctionnent avec BearLab AI sans modification. Vous devez simplement modifier l'`base_url` et fournir votre clé API BearLab AI. BearLab AI expose deux protocoles sur la même passerelle :

- **Compatible OpenAI** — L'URL de base est `<domain>/v1`, le point de terminaison est `/v1/chat/completions`.
- **Anthropic** — L'URL de base est la racine du domaine, le point de terminaison est `/v1/messages`.

<Note>
  Sélectionnez l'URL de base en fonction de votre région :

  - **États-Unis / Europe** — `https://bearlab.ai`
  - **Asie-Pacifique** — `https://bearlab.space`

  L'authentification est une clé API Bearer dans l'en-tête `Authorization`. Les SDK l'ajoutent automatiquement lorsque vous transmettez `api_key`. N'encodez jamais une vraie clé en dur dans le code source. Utilisez plutôt une variable d'environnement.
</Note>

## SDK OpenAI

Définissez `base_url` sur `<domain>/v1`. Les identifiants de modèle proviennent de la [liste des modèles](/fr/guides/models). Par exemple : `gpt-5.5`, `claude-sonnet-4-6` ou `deepseek-v4-pro`.

<CodeGroup>
```python Python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["BEARLAB_API_KEY"],   # sk-...
    base_url="https://bearlab.ai/v1",          # Asia-Pacific: https://bearlab.space/v1
)

resp = client.chat.completions.create(
    model="gpt-5.5",
    messages=[{"role": "user", "content": "Hello"}],
)
print(resp.choices[0].message.content)
```

```javascript Node.js
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.BEARLAB_API_KEY,        // sk-...
  baseURL: "https://bearlab.ai/v1",            // Asia-Pacific: https://bearlab.space/v1
});

const resp = await client.chat.completions.create({
  model: "gpt-5.5",
  messages: [{ role: "user", content: "Hello" }],
});
console.log(resp.choices[0].message.content);
```
</CodeGroup>

## SDK Anthropic

Définissez `base_url` sur la **racine du domaine** (sans le suffixe `/v1`). Le SDK Anthropic ajoute `/v1/messages` automatiquement. Utilisez un ID de modèle Claude depuis la [liste des modèles](/fr/guides/models), comme `claude-opus-4-7` ou `claude-sonnet-4-6`. `max_tokens` est un paramètre obligatoire de l'API Anthropic Messages.

<CodeGroup>
```python Python
import os
from anthropic import Anthropic

client = Anthropic(
    api_key=os.environ["BEARLAB_API_KEY"],   # sk-...
    base_url="https://bearlab.ai",             # Asia-Pacific: https://bearlab.space
)

msg = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello"}],
)
print(msg.content[0].text)
```

```javascript Node.js
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic({
  apiKey: process.env.BEARLAB_API_KEY,        // sk-...
  baseURL: "https://bearlab.ai",               // Asia-Pacific: https://bearlab.space
});

const msg = await client.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  messages: [{ role: "user", content: "Hello" }],
});
console.log(msg.content[0].text);
```
</CodeGroup>

<Warning>
  Chaque ID de modèle BearLab AI cible un protocole spécifique. Les modèles Claude sont accessibles via le SDK Anthropic ; les modèles au format OpenAI sont accessibles via le SDK OpenAI. Si une requête retourne une erreur « modèle non trouvé » ou « incompatibilité de point de terminaison », vérifiez que l'ID du modèle supporte le protocole que vous appelez. Consultez la [liste des modèles](/fr/guides/models).
</Warning>

<Tip>
  N'importe quel outil acceptant une URL de base OpenAI personnalisée (LobeChat, Chatbox, Continue, Zed, Immersive Translate) fonctionne de la même façon : définissez l'URL de base sur `https://bearlab.ai/v1` (États-Unis / Europe) ou `https://bearlab.space/v1` (Asie-Pacifique) et collez votre clé.
</Tip>
