所有请求都通过 API Key 鉴权。密钥在 [控制台](https://bearlab.ai) 的「令牌 / API Keys」页创建，形如 `sk-xxxxxxxx`。

<Note>
  亚太区域用户:请改用专用域名 `https://bearlab.in`，而非 `https://bearlab.ai`。详见 [服务区域](/zh-my/guides/availability)。
</Note>

## 如何传递密钥

两套协议的鉴权头不同：

| 协议 | 请求头 |
| --- | --- |
| OpenAI 兼容 | `Authorization: Bearer YOUR_API_KEY` |
| Anthropic 兼容 | `x-api-key: YOUR_API_KEY` |

<CodeGroup>
```bash OpenAI 兼容
curl https://bearlab.ai/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" ...
```

```bash Anthropic 兼容
curl https://bearlab.ai/v1/messages \
  -H "x-api-key: YOUR_API_KEY" ...
```
</CodeGroup>

## 密钥管理建议

- **一处一密钥**：为不同项目 / 客户端分别创建密钥，方便单独停用和统计用量。
- **设置额度上限**：可在创建时给单个密钥设定可用额度，降低泄露风险。
- **不要硬编码**：用环境变量注入，不要写进代码仓库。
- **泄露即吊销**：怀疑泄露时，在控制台删除该密钥并新建。

## 常见鉴权错误

| 现象 | 原因 |
| --- | --- |
| `401 Unauthorized` | 密钥错误、已删除，或鉴权头写错（Bearer / x-api-key 混用）。 |
| `403 Forbidden` | 密钥被禁用、余额不足，或无权访问该模型 / 分组；最常见的权限问题是模型需要实名认证。 |

更多状态码见 [错误处理](/zh-my/guides/errors)。
