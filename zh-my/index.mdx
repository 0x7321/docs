title: "BearLab AI"
description: "一个密钥搞定所有顶级对话模型。OpenAI、Anthropic、Google、DeepSeek 等通过单一 API 接入。"
---

BearLab AI (bearlab.ai) 将来自数十个提供商的对话模型整合到一个 API 中。创建一个密钥，设置一个 `base_url`，即可调用来自 OpenAI、Anthropic、Google、DeepSeek 及其他厂商的最新模型，无需分别向各家厂商注册和充值。

该接口兼容于 OpenAI 和 Anthropic 协议，因此现有代码通常只需更改一行 `base_url` 就能使用。

## 地域 base URL

选择您所在地区的 base URL，在两者之间使用相同的 API 密钥。

| 地区 | API 域名 | OpenAI 兼容 | Anthropic 兼容 |
| --- | --- | --- | --- |
| 美国 / 欧洲 | `https://bearlab.ai` | `https://bearlab.ai/v1/chat/completions` | `https://bearlab.ai/v1/messages` |
| 亚太地区 | `https://bearlab.space` | `https://bearlab.space/v1/chat/completions` | `https://bearlab.space/v1/messages` |

<Note>
  亚太地区的用户应使用专用域名 `https://bearlab.space` 而不是 `https://bearlab.ai`。您的 `sk-...` 密钥在两个域名上的功能相同。详见 [服务可用性](/zh-my/guides/availability)。
</Note>

## 两种协议，一个密钥

- **OpenAI 兼容** — `POST <base_url>/v1/chat/completions`，使用 `Authorization: Bearer sk-...` 进行身份验证。
- **Anthropic 兼容** — `POST <base_url>/v1/messages`，发送 `x-api-key: sk-...` 和 `anthropic-version: 2023-06-01`。

<Tabs>
  <Tab title="OpenAI 兼容 (美国 / 欧洲)">
    ```bash
    curl https://bearlab.ai/v1/chat/completions \
      -H "Authorization: Bearer sk-..." \
      -H "Content-Type: application/json" \
      -d '{
        "model": "gpt-5.5",
        "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
      }'
    ```
  </Tab>

  </Tab>
  <Tab title="Anthropic 兼容 (美国 / 欧洲)">
