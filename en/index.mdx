BearLab AI (bearlab.ai) brings conversational models from dozens of providers under one API. Create one key, set one `base_url`, and call the latest models from OpenAI, Anthropic, Google, DeepSeek, and others without registering and topping up separately with each vendor.

The interface is compatible with both the OpenAI and Anthropic protocols, so existing code usually works with a one-line `base_url` change.

## Regional base URLs

Pick the base URL for your region and keep the same API key across both.

| Region | API domain | OpenAI-compatible | Anthropic-compatible |
| --- | --- | --- | --- |
| US / EU | `https://bearlab.ai` | `https://bearlab.ai/v1/chat/completions` | `https://bearlab.ai/v1/messages` |
| Asia-Pacific | `https://bearlab.space` | `https://bearlab.space/v1/chat/completions` | `https://bearlab.space/v1/messages` |

<Note>
  Asia-Pacific users should use the dedicated domain `https://bearlab.space` instead of `https://bearlab.ai`. Your `sk-...` key works the same on both. See [Service Availability](/en/guides/availability).
</Note>

## Two protocols, one key

- **OpenAI-compatible** — `POST <base_url>/v1/chat/completions`, authenticate with `Authorization: Bearer sk-...`.
- **Anthropic-compatible** — `POST <base_url>/v1/messages`, send `x-api-key: sk-...` and `anthropic-version: 2023-06-01`.

<Tabs>
  <Tab title="OpenAI-compatible (US / EU)">
    ```bash
    curl https://bearlab.ai/v1/chat/completions \
      -H "Authorization: Bearer sk-..." \
      -H "Content-Type: application/json" \
      -d '{
        "model": "gpt-5.5",
        "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
      }'
    ```
  </Tab>
