This page lists the models your API key can call, grouped by family. Pass a model's exact `id` in the `model` field of a chat request.

<Note>
  Asia-Pacific users: use the dedicated domain `https://bearlab.space` instead of `https://bearlab.ai`. See [Service Availability](/en/guides/availability).
</Note>

## How to read this catalog

The catalog you see is keyed to **your token's group**. Each API key belongs to one model group, and a model only appears in your list if your group is allowed to call it. The models below are the current production line-up; the exact subset visible to your key depends on your group.

| Concept | What it means |
| --- | --- |
| **Model `id`** | The exact string you send in `model`. Case-sensitive, copy it verbatim. |
| **Model group** | The access tier your key belongs to. Determines which models you can call and at what price. |
| **Endpoint type** | Which protocol a model accepts: OpenAI-compatible, Anthropic, or Gemini. |

## Fetch the live list

`/v1/models` returns the models your key can call, in OpenAI SDK format. It requires your API key. The response format follows the format implied by your request headers (OpenAI by default; Anthropic with `x-api-key` + `anthropic-version`; Gemini with `x-goog-api-key`).

<Tabs>
  <Tab title="US / EU (bearlab.ai)">
    ```bash
    curl https://bearlab.ai/v1/models \
      -H "Authorization: Bearer sk-..."
    ```
  </Tab>
  <Tab title="Asia-Pacific (bearlab.space)">
    ```bash
    curl https://bearlab.space/v1/models \
      -H "Authorization: Bearer sk-..."
    ```
  </Tab>
</Tabs>

Response (truncated):

```json
{
  "object": "list",
  "data": [
    {
      "id": "claude-opus-4-8",
      "object": "model",
      "created": 1626777600,
      "owned_by": "openai",
      "supported_endpoint_types": ["openai"]
    }
  ]
}
```

| Field | Description |
| --- | --- |
| `id` | Model identifier; pass this in the `model` field of a chat request |
| `object` | Always `model` |
| `created` | Release time (Unix seconds) |
| `owned_by` | Nominal owner string |
| `supported_endpoint_types` | Which endpoints accept this model: `openai`, `anthropic`, and/or `gemini` |

## Calling a model

Send the exact `id` in the `model` field. Both example domains are shown; pick the one for your region.

<Tabs>
  <Tab title="US / EU (bearlab.ai)">
    ```bash
    curl https://bearlab.ai/v1/chat/completions \
      -H "Authorization: Bearer sk-..." \
      -H "Content-Type: application/json" \
      -d '{
        "model": "claude-sonnet-4-6",
        "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
      }'
    ```
  </Tab>
  <Tab title="Asia-Pacific (bearlab.space)">
    ```bash
    curl https://bearlab.space/v1/chat/completions \
      -H "Authorization: Bearer sk-..." \
      -H "Content-Type: application/json" \
      -d '{
        "model": "claude-sonnet-4-6",
        "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
      }'
    ```
  </Tab>
</Tabs>
