---
title: "Models & Pricing API"
description: "A public, unauthenticated endpoint listing all available conversational models and their published prices, following the OpenRouter Providers specification."
---

`/v1/models/pricing` is a **public, unauthenticated** aggregation endpoint that returns every conversational model currently available along with its published price. The response follows the [OpenRouter Providers specification](https://openrouter.ai/docs/guides/community/for-providers), so it can be ingested by OpenRouter, third-party price aggregators, or any custom client.

<Note>
  Asia-Pacific users: use `https://bearlab.in` as the endpoint instead of `bearlab.ai`. See [Service Availability](/guides/availability).
</Note>

<Note>
  This is the "published price" entry point and carries no user identity. Prices represent the `default` group in an anonymous context. To see the price an authenticated user actually pays (group, tier discount, etc.), use the pricing page in the console.
</Note>

## When to use it

| Use case | Description |
| --- | --- |
| Price aggregation | OpenRouter and other aggregators scraping the catalog and published prices |
| Client display | Apps, bots, and SDKs that render the model list |
| Monitoring | Tracking catalog or price changes over time |

## Endpoint

| Item | Value |
| --- | --- |
| **URL** | `https://bearlab.ai/v1/models/pricing` |
| **Method** | `GET` |
| **Auth** | None (public) |
| **Rate limit** | Strict per-IP critical rate limit |
| **Caching** | 60s in-process cache; `Cache-Control: public, max-age=60` |

### Request

```bash
curl https://bearlab.ai/v1/models/pricing
```

## Response structure

The top level is always wrapped in a `data` array:

```json
{
  "data": [
    {
      "id": "gpt-4o",
      "name": "GPT-4o",
      "created": 1715558400,
      "input_modalities": ["text"],
      "output_modalities": ["text"],
      "context_length": 128000,
      "max_output_length": 16384,
      "pricing": {
        "prompt": "0.0000025",
        "completion": "0.00001",
        "request": "0",
        "input_cache_read": "0.00000125"
      },
      "supported_features": ["tools", "json_mode", "structured_outputs", "logprobs"]
    }
  ]
}
```

### Model fields

| Field | Type | Description |
| --- | --- | --- |
| `id` | `string` | Unique model identifier; use it when calling `/v1/chat/completions` |
| `name` | `string` | Display name; falls back to `id` |
| `created` | `int64` | Release time (Unix seconds) |
| `input_modalities` / `output_modalities` | `string[]` | At least `text` |
| `context_length` | `int` | Maximum context window in tokens |
| `max_output_length` | `int` | Maximum output tokens per response |
| `pricing` | `Pricing` | Pricing object; see below |
| `pricing_tiers` | `Pricing[]` | Tiered pricing (at most one, by context-length threshold) |
| `supported_features` | `string[]` | Capability flags |

### Pricing fields

All prices are **USD strings**; token-class fields are priced per token.

| Field | Unit | Description |
| --- | --- | --- |
| `prompt` | USD / input token | Input unit price |
| `completion` | USD / output token | Output unit price |
| `request` | USD / request | Flat per-request fee (usually `"0"` for chat models) |
| `internal_reasoning` | USD / reasoning token | Reasoning token price |
| `input_cache_read` | USD / token | Cache-hit read price (usually lower) |
| `input_cache_write` | USD / token | Cache write price |
| `web_search` | USD / query | Built-in web search tool fee |

<Note>
  A value of `"0"` may mean "free" or "not applicable to this model". Combine `supported_features` to infer capability rather than relying on pricing alone.
</Note>

### `supported_features`

| Value | Meaning |
| --- | --- |
| `tools` | Function calling / tool use |
| `reasoning` | Returns reasoning tokens |
| `json_mode` | JSON mode output |
| `structured_outputs` | Schema-constrained structured outputs |
| `logprobs` | Token log probabilities |
| `web_search` | Built-in web search tool |

## Caching and update cadence

| Aspect | Behavior |
| --- | --- |
| Server cache | 60s in-process; admin updates take effect within 1 minute |
| Client cache | `Cache-Control: public, max-age=60` |
| Recommended polling | At most once every 5 minutes; aggressive polling is rate-limited |

## Comparison with other endpoints

| Endpoint | Purpose | Auth |
| --- | --- | --- |
| `/v1/models/pricing` | Public pricing in OpenRouter format | Public |
| `/v1/models` | OpenAI-SDK-compatible model id list | Token required |
