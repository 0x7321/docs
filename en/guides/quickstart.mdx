Three steps to call a model: create a key, point your client at the right base URL, then send your first chat request.

<Note>
  Pick the base URL for your region. **US / EU:** `https://bearlab.ai`. **Asia-Pacific:** `https://bearlab.space`. Everything else (keys, models, request format) is identical across regions. See [Service Availability](/en/guides/availability).
</Note>

<Steps>
  <Step title="Sign up and top up">
    Register at the [console](https://bearlab.ai) and add funds on the Wallet page. New users can start with a small top-up to test.
  </Step>
  <Step title="Create an API key">
    Go to Dashboard, API Tokens, click Add Token, and copy the `sk-...` key. The key is shown in full only once. You pass it as `Authorization: Bearer sk-...` on every request.
  </Step>
  <Step title="Set your base URL">
    Use the base URL that matches your region and protocol:

    | Protocol | US / EU | Asia-Pacific |
    | --- | --- | --- |
    | OpenAI-compatible | `https://bearlab.ai/v1` | `https://bearlab.space/v1` |
    | Anthropic | `https://bearlab.ai/v1/messages` | `https://bearlab.space/v1/messages` |
  </Step>
  <Step title="Make your first request">
    Send the example below. Replace `sk-...` with the key you just created.
  </Step>
</Steps>

## Examples

The OpenAI-compatible endpoint is `POST /v1/chat/completions`; the Anthropic endpoint is `POST /v1/messages`. Both authenticate with `Authorization: Bearer sk-...`. The `model` value must be one returned by the [models list](/en/guides/models) for your key — the examples below use models available to this account.

### OpenAI-compatible

<CodeGroup>
```bash US / EU (bearlab.ai)
curl https://bearlab.ai/v1/chat/completions \
  -H "Authorization: Bearer sk-..." \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.4",
    "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
  }'
```

```bash Asia-Pacific (bearlab.space)
curl https://bearlab.space/v1/chat/completions \
  -H "Authorization: Bearer sk-..." \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.4",
    "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
  }'
```
</CodeGroup>

### Anthropic

The Anthropic endpoint expects `max_tokens` and the `anthropic-version` header. You can authenticate with `Authorization: Bearer sk-...` (the gateway also accepts `x-api-key: sk-...`).

<CodeGroup>
```bash US / EU (bearlab.ai)
curl https://bearlab.ai/v1/messages \
  -H "Authorization: Bearer sk-..." \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4-6",
    "max_tokens": 256,
    "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
  }'
```

```bash Asia-Pacific (bearlab.space)
curl https://bearlab.space/v1/messages \
  -H "Authorization: Bearer sk-..." \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4-6",
    "max_tokens": 256,
    "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
  }'
```
</CodeGroup>

### Python (OpenAI SDK)

Point the SDK at the `/v1` base URL for your region. The SDK appends `/chat/completions` automatically.

<CodeGroup>
```python US / EU (bearlab.ai)
from openai import OpenAI

client = OpenAI(
    api_key="sk-...",
    base_url="https://bearlab.ai/v1",
)

resp = client.chat.completions.create(
    model="gpt-5.4",
    messages=[{"role": "user", "content": "Introduce yourself in one sentence"}],
)
print(resp.choices[0].message.content)
```

```python Asia-Pacific (bearlab.space)
from openai import OpenAI

client = OpenAI(
    api_key="sk-...",
    base_url="https://bearlab.space/v1",
)

resp = client.chat.completions.create(
    model="gpt-5.4",
    messages=[{"role": "user", "content": "Introduce yourself in one sentence"}],
)
print(resp.choices[0].message.content)
```
</CodeGroup>
