[Codex CLI](https://github.com/openai/codex) is OpenAI's official command-line coding agent. You connect it to BearLab AI by adding a custom provider that points at the BearLab AI gateway.

<Warning>
  Codex uses the OpenAI **Responses API** (`wire_api = "responses"`), an OpenAI-proprietary protocol that **only works with OpenAI-family models** (`gpt-*` / `gpt-oss-*`). Claude, Gemini, GLM and other non-OpenAI models cannot be driven through Codex — for those, use [Claude Code](/en/integrations/claude-code), [Cursor](/en/integrations/cursor), or [Cline](/en/integrations/cline) instead.
</Warning>

## Base URLs

Codex talks to the OpenAI-compatible endpoint, so `base_url` ends in `/v1`. Pick the region closest to you:

| Region | `base_url` |
| --- | --- |
| US / EU | `https://bearlab.ai/v1` |
| Asia-Pacific | `https://bearlab.space/v1` |

## Recommended Models

Use an OpenAI-family model ID exactly as it appears in your [models list](/en/guides/models). The available OpenAI-family IDs include:

| Model | ID | Best For |
| --- | --- | --- |
| GPT-5.5 | `gpt-5.5` | Latest general model, strong default for Codex |
| GPT-5.4 | `gpt-5.4` | Balanced quality and speed |
| GPT-5.4-mini | `gpt-5.4-mini` | Lightweight and fast, sub-tasks and interactive edits |
| GPT-5.2 | `gpt-5.2` | Reliable workhorse for everyday changes |
| GPT-5 | `gpt-5` | Stable baseline, broad compatibility |

<Note>
  Model IDs are case-sensitive and must match the [models endpoint](/en/guides/models) verbatim. Run a quick listing to confirm what your key can see before configuring Codex.
</Note>

## Configuration

Edit `~/.codex/config.toml` and add a provider pointing at BearLab AI.

<Tabs>
  <Tab title="US / EU">
    ```toml
    model = "gpt-5.5"
    model_provider = "bearlab"

    # Third-party providers must disable response storage
    disable_response_storage = true

    [model_providers.bearlab]
    name = "BearLab AI"
    base_url = "https://bearlab.ai/v1"
    env_key = "BEARLAB_API_KEY"
    wire_api = "responses"
    ```
  </Tab>
  <Tab title="Asia-Pacific">
    ```toml
    model = "gpt-5.5"
    model_provider = "bearlab"

    # Third-party providers must disable response storage
    disable_response_storage = true

    [model_providers.bearlab]
    name = "BearLab AI"
    base_url = "https://bearlab.space/v1"
    env_key = "BEARLAB_API_KEY"
    wire_api = "responses"
    ```
  </Tab>
</Tabs>

Export your API key and run:

```bash
export BEARLAB_API_KEY="sk-..."
codex
```

<Note>
  Use the OpenAI-compatible address ending in `/v1`. `wire_api = "responses"` and `disable_response_storage = true` are required for a third-party OpenAI-format provider. Authentication is a Bearer key (`sk-...`) that Codex sends in the `Authorization` header via `env_key`. `model` must be an OpenAI-family ID that exists in your [models list](/en/guides/models).
</Note>
