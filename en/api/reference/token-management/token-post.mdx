---
title: "Create Token"
description: "POST /api/token/"
---

## Request Headers

| Header | Required | Description |
| --- | --- | --- |
| `Authorization` | ✓ | `Bearer YOUR_ACCESS_TOKEN`, account access token (console login session, not `sk-` API Key) |
| `Content-Type` | ✓ | `application/json` |

Endpoint: `POST /api/token/`

Create a new API Key. The key is generated by the server and **not returned in the response** — you must copy and save it once in the console.

## Request Body

| Field | Required | Type | Description |
| --- | --- | --- | --- |
| `name` | ✓ | string | Token name (≤ 50 characters) |
| `expired_time` | ✓ | integer | Expiration time in Unix seconds; `-1` means never expires |
| `remain_quota` | ✓ | integer | Initial quota; use with `unlimited_quota: true` for unlimited quota |
| `unlimited_quota` | ✓ | boolean | Whether unlimited quota is enabled |
| `model_limits_enabled` |  | boolean | Whether model whitelist is enabled |
| `model_limits` |  | string | Model whitelist (comma-separated; empty string means no restriction) |
| `allow_ips` |  | string \| null | IP whitelist (empty string / `null` means no restriction) |
| `group` |  | string | Group, taken from the key of `GET /api/user/self/groups`; empty string means not specified |
| `cross_group_retry` |  | boolean | Whether to retry across groups (only valid for `auto` group) |

```bash
curl -X POST https://bearlab.ai/api/token/ \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"my-token","expired_time":-1,"remain_quota":-1,"unlimited_quota":true,"model_limits_enabled":false,"group":""}'
```

```json
{
  "success": true,
  "message": ""
}
```

<Note>
Source: new-api source code `controller/token.go` `AddToken`. Successful response contains only `{ success, message }`, **no `data` field**, and does not return the full key. Token struct has no `exclude_ips` field. After creation, use `GET /api/token/` to view (the `key` is masked).
</Note>
