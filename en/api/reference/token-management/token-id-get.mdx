---
title: "Get Token Details"
description: "GET /api/token/{id}"
---

## Request Headers

| Header | Required | Description |
| --- | --- | --- |
| `Authorization` | ✓ | `Bearer YOUR_ACCESS_TOKEN`, account access token (console login session, not an `sk-` API Key) |

Endpoint: `GET /api/token/{id}`

Retrieve the configuration details of a single token by Token ID. The `key` field in the response is masked; to retrieve the **complete plaintext Key**, use `POST /api/token/{id}/key`.

## Path Parameters

| Parameter | Required | Description |
| --- | --- | --- |
| `id` | ✓ | Token ID, obtained from `data.items[].id` in `GET /api/token/` |

```bash
curl https://bearlab.ai/api/token/10665 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```json Response
{

  "message": "",
  "data": {

    "user_id": 42,
    "key": "VrqY**********tE4I",
    "status": 1,
    "name": "my-token",
    "created_time": 1781977779,
    "accessed_time": 1781977779,
    "expired_time": -1,
    "remain_quota": 0,
```

The response is a masked Token object with complete details; field meanings are explained in [Token List](/en/api/reference/token-management/token-get).

<Note>
Source: new-api source code `controller/token.go` `GetToken`, response `data` is `buildMaskedTokenResponse(token)` (`key` is masked). Retrieving the complete plaintext Key is a separate endpoint `POST /api/token/{id}/key` (`GetTokenKey`).
</Note>
