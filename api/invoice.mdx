<Note>
發票功能需平台已啟用。分頁使用 `page_size` / `total`，與工單（`size` / `total_count`）不同。
</Note>

## 我的發票 `GET /api/invoice/self`

| 參數 | 說明 |
| --- | --- |
| `page` | 頁碼 |
| `page_size` | 每頁數量 |

```bash
curl "https://bearlab.ai/api/invoice/self?page=1&page_size=10" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```json 回應
{
  "data": {
    "items": [],
    "page": 1,
    "page_size": 5,
    "total": 0
  }
}
```

## 可開票訂單 `GET /api/invoice/available_orders`

返回可申請發票的已付款訂單。無符合訂單時返回空陣列。

```bash
curl https://bearlab.ai/api/invoice/available_orders \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```json 回應
{ "data": [], "success": true }
```

---

## 最新發票資訊 `GET /api/invoice/latest_info`

返回上次提交的發票抬頭資訊。首次申請前為 `null`。

```bash
curl https://bearlab.ai/api/invoice/latest_info \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```json 回應（無歷史記錄）
{ "data": null, "success": true }
```

```json 回應（有歷史記錄）
{
  "data": {
    "company_name": "範例科技有限公司",
    "tax_number": "91310000XXXXXXXX",
    "email": "billing@example.com"
  },
  "success": true
}
```

## 提交發票申請 `POST /api/invoice/submit`

```bash
curl -X POST https://bearlab.ai/api/invoice/submit \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "order_ids": [6701],
    "company_name": "範例科技有限公司",
    "tax_number": "91310000XXXXXXXX",
    "email": "billing@example.com"
  }'
```

| 欄位 | 必填 | 說明 |
| --- | --- | --- |
| `order_ids` | ✓ | 訂單 ID 陣列，來自可開票訂單 |
| `company_name` | ✓ | 發票抬頭 |
| `tax_number` | ✓ | 稅號 |
| `email` | ✓ | 發票接收郵件 |

## 發票下載連結 `GET /api/invoice/download-link/{id}`

| 路徑參數 | 說明 |
| --- | --- |
| `id` | 發票 ID |

```bash
curl https://bearlab.ai/api/invoice/download-link/123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```json 回應
{
  "data": { "url": "https://..." },
  "success": true
}
```
