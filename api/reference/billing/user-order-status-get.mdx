## 请求头

| 请求头 | 必填 | 说明 |
| --- | --- | --- |
| `Authorization` | ✓ | `Bearer YOUR_ACCESS_TOKEN`，账号访问令牌 |

接口：`GET /api/user/order/status`

按平台订单号查询充值是否完成。前端创建订单后会用该接口轮询，直到 `data` 为 `true`。

| 查询参数 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| `trade_no` | ✓ | string | `POST /api/user/order` 返回的 `data.trade_no` |

```bash
curl "https://bearlab.ai/api/user/order/status?trade_no=USR3930NOxxxxxxxx1782010000" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```json 回应
{
  "data": true,
  "message": "",
  "success": true
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `data` | boolean | `true` 表示订单已完成；`false` 表示仍未完成或未确认到账 |

<Note>
来源：2026-06-21 前端调用链实测。轮询间隔可参考前端实现：约 3 秒一次，成功后停止。
</Note>
