所有端點均需 `Authorization: Bearer YOUR_API_KEY` 標頭，基底 URL 為 `https://bearlab.ai`。

---

## 取得當前使用者 `GET /api/user/self`

返回登入帳號的完整資訊（50+ 欄位）。

<Warning>
`user.id` 不存在於此回應中。`user_id` 僅在 `/api/user/checkin-history` 和 `PUT /api/token/` 回應中可見。回應中含有 `access_token` 明文欄位（即當前 Bearer token 本身）及 `DeletedAt` 欄位（GORM 軟刪除欄位，正常情況下為 `null`）。
</Warning>

```bash
curl https://bearlab.ai/api/user/self \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## 更新使用者資訊 `PUT /api/user/self`

更新顯示名稱、語言、頭像等基本設定。

```bash
curl -X PUT https://bearlab.ai/api/user/self \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"display_name":"My Name","language":"zh-Hant"}'
```

## 取得存取令牌 `GET /api/user/token`

<Warning>
**破壞性操作**：每次呼叫都會重新生成 `access_token`，舊令牌立即失效。此令牌用於系統層面鑑權，與 API Key (`/api/token/`) 不同。
</Warning>

```bash
curl https://bearlab.ai/api/user/token \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```json 回應
{
  "data": "DmGxCrprnCE7dHTFUGK2dCZ/3T1U+hA=",
  "success": true
}
```

## 登出 `GET /api/user/logout`

使當前 session 失效。

```bash
curl https://bearlab.ai/api/user/logout \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```json 回應
{ "data": null, "message": "", "success": true }
```

## 平台狀態 `GET /api/status`

公開端點，無需鑑權。返回平台基本設定。

```bash
curl https://bearlab.ai/api/status
```

```json 回應
{
  "data": {
    "enable_checkin": false,
    "exchange_rate": 1,
    "api_address": "https://bearlab.ai",
    "default_currency_zh": "CNY",
    "version": "v6.3.13"
  }
}
```

## 使用者等級 `GET /api/level`

## KYC 認證狀態 `GET /api/user/verify/self`

```json 回應
{
  "data": {
    "user_id": 3930,
    "type": "",
    "status": "",
    "created_at": "2026-06-21T06:47:15.368+08:00",
    "updated_at": "2026-06-21T06:47:15.368+08:00"
  }
}
```

| `type` 值 | 說明 |
| --- | --- |
| `""` | 未認證 |
| `"personal"` | 個人認證 |
| `"enterprise"` | 企業認證 |
| `"passport"` | 護照認證 |

| `status` 值 | 說明 |
| --- | --- |
| `""` | 未提交 |
| `"PROCESSING"` | 審核中 |
| `"SUCCESS"` | 已通過 |
| `"FAIL"` | 已拒絕 |

## 簽到記錄 `GET /api/user/checkin-history`

<Note>
`user_id` 在此端點可見，而 `/api/user/self` 不返回 `id` 欄位。
</Note>

```json 回應
{
  "data": [
    {
      "id": 15987,
      "user_id": 3930,
      "check_in_at": "2026-06-20T00:00:00+08:00",
      "quota": "0.0128"
    }
  ]
}
```
