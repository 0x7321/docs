BearLab AI पर प्रत्येक मॉडल एक या अधिक **अपस्ट्रीम प्रदाताओं** (विक्रेताओं) द्वारा परोसा जाता है। एक एकल मॉडल नाम के पीछे, गेटवे आपके अनुरोध को एक स्वस्थ प्रदाता नोड तक पहुंचाता है; यदि एक विफल हो जाता है या समय समाप्त हो जाता है, तो यह स्वचालित रूप से दूसरे को पुनः प्रयास करता है, आपके कोड के लिए पारदर्शी। प्रदाताओं को **रूटिंग समूहों** में संगठित किया जाता है, और आपका टोकन यह तय करता है कि यह कौन से समूहों से आकर्षित कर सकता है।

<Note>
BearLab AI एक एकीकृत गेटवे है जो New API पर निर्मित है। नीचे दिया गया प्रदाता कैटलॉग इस खाते के लाइव मूल्य निर्धारण डेटा से उत्पन्न होता है, इसलिए यह बिल्कुल वही विक्रेताओं और मॉडल परिवारों को दर्शाता है जो आपकी कुंजी के साथ पहुंचने योग्य हैं।
</Note>

## आधार URL

अपने निकटतम क्षेत्र को चुनें। दोनों क्षेत्र समान मॉडल और समान प्रदाता रूटिंग को प्रदर्शित करते हैं।

| क्षेत्र | OpenAI-संगत आधार URL | Anthropic आधार URL |
| --- | --- | --- |
| अमेरिका / EU | `https://bearlab.ai/v1` | `https://bearlab.ai` (संदेश `/v1/messages` पर) |
| एशिया-प्रशांत | `https://bearlab.space/v1` | `https://bearlab.space` (संदेश `/v1/messages` पर) |

प्रमाणीकरण हर जगह समान है: अपनी API कुंजी को Bearer टोकन के रूप में भेजें।

```
Authorization: Bearer sk-...
```

## कवर किए गए प्रदाता

प्रत्येक प्रदाता एक अपस्ट्रीम विक्रेता रूट के अनुरूप है। यह तालिका लाइव मॉडल-से-विक्रेता मैपिंग से बनाई गई है; गणना उस प्रदाता के तहत वर्तमान में मूल्य निर्धारित मॉडल वेरिएंट की संख्या है।

| प्रदाता | अपस्ट्रीम विक्रेता | मॉडल परिवार | वेरिएंट | प्रोटोकॉल |
| --- | --- | --- | --- | --- |
| Anthropic | Anthropic (Bedrock / Kiro रिलेज़ सहित) | Claude (Opus, Sonnet, Haiku) | 30 | OpenAI + Anthropic |
| Google | Google Gemini | Gemini 2.5 / 3 / 3.1 / 3.5 (Pro, Flash, छवि, CLI वेरिएंट) | 79 | OpenAI + Gemini |
| OpenAI | OpenAI | GPT-5.x, GPT-image-2 | 23 | OpenAI |
| xAI | xAI | Grok 4 / 4.1 / 4.2 / 4.3 | 10 | OpenAI |
| Alibaba Bailian (एग्रीगेटर) | Alibaba Cloud Bailian | Qwen, GLM, DeepSeek, Kimi, MiniMax | 18 | OpenAI + Anthropic |

<Note>
कुछ मॉडल परिवार एक से अधिक प्रदाता के माध्यम से पहुंचने योग्य हैं। उदाहरण के लिए, राष्ट्रीय मॉडल (Qwen / GLM / DeepSeek / Kimi / MiniMax) सभी अलग-अलग मूल विक्रेता रूट के बजाय **Alibaba Bailian** एग्रीगेटर के माध्यम से परोसे जाते हैं। गेटवे आपके लिए एक उपलब्ध रूट चुनता है, इसलिए आपको शायद ही कभी एक विशिष्ट प्रदाता का नाम देने की आवश्यकता होती है।
</Note>

<Warning>
समूह अभिगम आपकी कुंजी से बंधा हुआ है। एक अनुरोध में एक समूह या प्रदाता का नाम देना जिसके लिए आपके पास अनुमति नहीं है, इसे अनलॉक नहीं करेगा — गेटवे केवल उसी के भीतर रूट करता है जिसे आपका टोकन करने के लिए अधिकृत है।
</Warning>

## एक टोकन में रूटिंग कॉन्फ़िगर करना

जब आप एक टोकन बनाते या संपादित करते हैं तो आप इसकी डिफ़ॉल्ट रूटिंग सेट करते हैं:

- **प्रदाता / समूह सूची**: अपनी कुंजी द्वारा पहुंचने योग्य सभी चीजों में रूट करने के लिए खाली छोड़ दें (अनुशंसित); इसे प्रतिबंधित करने के लिए एक एकल प्रविष्टि सेट करें; क्रम में या क्रम रणनीति द्वारा उन्हें आजमाने के लिए कई (अल्पविराम-अलग) सेट करें।
- **केवल प्रदाता मोड**: बंद (डिफ़ॉल्ट) सूचीबद्ध प्रदाताओं को प्राथमिकता देता है लेकिन अन्य में वापस गिरता है; चालू सख्ती से उन्हीं तक सीमित है और यदि सभी विफल हो जाते हैं तो त्रुटि लौटाता है।
- **क्रम रणनीति**: मैनुअल (डिफ़ॉल्ट), लेटेंसी, मूल्य, या थ्रूपुट।

## अनुरोध के आधार पर ओवरराइड

हेडर के माध्यम से एक एकल अनुरोध के लिए अपने टोकन की डिफ़ॉल्ट सेटिंग्स को ओवरराइड करें:

| हेडर | प्रभाव |
| --- | --- |
| `X-Provider-Order` | इस अनुरोध के लिए उपयोग करने के लिए प्रदाता/समूह और क्रम |
| `X-Provider-Only` | `"true"` सख्त मोड सक्षम करता है (कोई फेलबैक नहीं) |
| `X-Provider-Sort` | क्रम रणनीति ओवरराइड करें (`latency` / `price` / `throughput`) |
| `X-Provider-Ignore` | छोड़ने के लिए प्रदाता, टोकन कॉन्फ़िग पर स्टैक किए गए |

> हेडर के माध्यम से अनुरोध किए गए प्रदाता और समूह अभी भी आपकी खाता अनुमतियों के अधीन हैं।

### उदाहरण

<CodeGroup>

```bash अमेरिका / EU (bearlab.ai)
curl https://bearlab.ai/v1/chat/completions \
  -H "Authorization: Bearer sk-..." \
  -H "Content-Type: application/json" \
  -H "X-Provider-Sort: latency" \
  -d '{
    "model": "gpt-5.5",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

```bash एशिया-प्रशांत (bearlab.space)
curl https://bearlab.space/v1/chat/completions \
  -H "Authorization: Bearer sk-..." \
  -H "Content-Type: application/json" \
  -H "X-Provider-Sort: latency" \
  -d '{
    "model": "gpt-5.5",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

</CodeGroup>

## लाइव प्रदाताओं की खोज करना

ऊपर दी गई सूची एक स्नैपशॉट है। यह देखने के लिए कि आपकी कुंजी अभी कौन से सटीक मॉडल तक पहुंच सकती है, मॉडल एंडपॉइंट को क्वेरी करें — गेटवे आपके प्रमाणीकरण हेडर से प्रोटोकॉल को स्वचालित रूप से समझता है:

<CodeGroup>

```bash OpenAI प्रोटोकॉल
curl https://bearlab.ai/v1/models \
  -H "Authorization: Bearer sk-..."
```

```bash Anthropic प्रोटोकॉल
curl https://bearlab.ai/v1/models \
  -H "x-api-key: sk-..." \
  -H "anthropic-version: 2023-06-01"
```

</CodeGroup>

प्रत्येक प्रविष्टि में `id`, `owned_by` और `supported_endpoint_types` शामिल हैं, इसलिए आप इसे कॉल करने से पहले बता सकते हैं कि दिया गया मॉडल कौन सी प्रोटोकॉल स्वीकार करता है। पूर्ण कैटलॉग के लिए [Models](/hi/guides/models) देखें और प्रति-मॉडल दरों के लिए [Pricing](/hi/guides/pricing) देखें।
