按用量付费。无订阅、无最低消费——每次请求都从您的账户余额扣除。

## 账单如何运作

您的账户持有以 BearLab AI 配额衡量的**信用额度**。每次请求根据您调用的模型和处理的 token 数量（或对于某些模型，按固定的按次价格）扣除配额。整个控制台显示的美元金额采用固定换算：

<Note>
**500,000 配额 = 1 美元**。本页面上的所有按 token 和按次图表均以美元表示，便于阅读。
</Note>

您任何请求的最终成本也受到请求路由所经过之[供应商群组](/zh-Hans/guides/providers)的**群组倍数**影响，[用户等级](/zh-Hans/guides/user-levels)可能会应用额外折扣。

## 两种账单方式

每个模型以两种方式之一计费，由其 `quota_type` 决定：

<CardGroup cols={2}>
  <Card title="按 token 付费" icon="coins">
    `quota_type = 0`。输入和输出 token 分别定价。涵盖对话/聊天模型——Claude、GPT-5.x、Gemini、Grok 和国内模型。
  </Card>
  <Card title="按次付费" icon="receipt">
    `quota_type = 1`。无论 token 数量如何，每次成功请求均收取固定价格。用于图像生成、CLI/按次套件和计量端点。
  </Card>
</CardGroup>

## 按 token 定价

对于按 token 计费的模型，BearLab AI 使用三个数字：

| 字段 | 含义 |
| --- | --- |
| `model_ratio` | 输入价格倍数。**每 100 万 token 的输入美元 = `model_ratio` × 2。** |
| `completion_ratio` | 相对于输入的输出倍数。**每 100 万 token 的输出美元 = 输入价格 × `completion_ratio`。** |
| `cache_ratio` | 缓存命中的输入 token 按 `cache_ratio` 的输入价格计费（大多数聊天模型：`0.1`，即 90% 折扣）。见[提示词缓存](/zh-Hans/guides/prompt-caching)。 |

Token 计数来自回应 `usage` 对象中的 `prompt_tokens` 和 `completion_tokens`。

<Note>
**成本公式（按 token 计费）：**
`cost = (input_tokens × input_price + output_tokens × input_price × completion_ratio) × group_ratio`
其中 `input_price = model_ratio × $2 / 100 万 token`。缓存输入 token 替代 `model_ratio × cache_ratio`。
</Note>

### 分层/上下文长度定价

某些模型——主要是通过阿里云路由的国内模型（DeepSeek、Qwen、GLM、Kimi、MiniMax）——使用**分层表达式**：按 token 比率随请求的输入长度而变化。例如，较长上下文的 Qwen 和 GLM 分层在超过 32K / 128K / 256K token 时收取较高的输入倍数。这些模型在控制台定价页面上显示其分层断点；账单比率根据请求的输入长度自动选择。

## 按次定价

对于 `quota_type = 1` 模型，每次成功请求收取固定 `model_price`（美元）。`model_ratio` 和 `completion_ratio` 不适用。群组倍数仍然适用。

| 类别 | 模型示例 | 按次价格 |
| --- | --- | --- |
| 图像生成 | `gpt-image-2` | $0.05 |
| 图像生成 (VIP) | `gpt-image-2-vip` | $0.065 |
| 图像生成 | `gemini-3-pro-image-preview` | $0.15 |
| Gemini CLI / 按次 | `gemini-2.5-pro-cli`, `gemini-3.1-pro-preview-cli` | $0.0001 |
| Gemini 计量分层 | `gemini-3-pro-preview-c1`, `gemini-2.5-pro-c1` | $0.0002 – $0.0003 |
| Kiro-Claude 按次 | `kiro-claude-opus-4-8`, `kiro-claude-sonnet-4-6` | $0.000305 – $0.00058 |

<Note>
按次成本就是 `model_price × group_ratio`。失败或遭拒的请求不计费。
</Note>

## 免费模型

如果您的密钥仅解析到**免费群组**，您看到的模型（例如 `claude-opus-4-7`、`deepseek-v4-pro`、`gpt-5.4-mini`、`claude-opus-4-8`）会以零成本路由到您的余额。控制台可能会显示名义上的 `model_price`，但这些模型的有效群组倍数是 **0**，因此不会扣除配额。见[模型指南](/zh-Hans/guides/models)了解您的密钥传回的完整免费模型列表。

## 群组倍数和折扣

同一模型根据您的密钥绑定到的[供应商群组](/zh-Hans/guides/providers)可能有不同的有效价格。您支付的价格等于模型的基本价格（按 token 或按次）乘以群组倍数。更高的[用户等级](/zh-Hans/guides/user-levels)可能解锁额外折扣。

## 充值

- 从控制台充值您的余额；已充值的配额立即可用。
- 对于企业账单或大额充值，[联系我们](mailto:support@bearlab.ai)。

## 余额警示

为避免中断，在控制台**个人资料 → 设置**下设定低余额阈值。当您的余额降至以下时，我们会通过电子邮件或应用内消息通知您。

## 高量方案

如需更高并发性或专属定价，[联系我们](mailto:support@bearlab.ai)。

## 退款

若要申请退款，请联系支持部门。注意：

- 如果退款导致您的账户等级下降，超额获利金额将被扣除。
- 已开立发票的充值不可退款。
- 收取最高 **5%** 的处理费。
- 退款将退回到原始付款方式。

如需正式发票，请联系支持部门并提供您的公司名称、税务 ID 和金额。
