对话补全是核心接口。提供 OpenAI、Anthropic、Gemini 三套兼容实现，共用同一密钥。

<Note>
  亚太区域用户:接口地址请改用 `https://bearlab.in`，而非 `bearlab.ai`。详见 [服务区域](/zh-Hans/guides/availability)。
</Note>

## 端点

| 协议 | 方法与路径 |
| --- | --- |
| OpenAI 兼容 | `POST https://bearlab.ai/v1/chat/completions` |
| Anthropic 兼容 | `POST https://bearlab.ai/anthropic/v1/messages` |
| Gemini 原生 | `base_url=https://bearlab.ai`，`api_version=v1beta`（Google genai SDK） |

## 基本请求

<Tabs>
  <Tab title="OpenAI 兼容">
    ```bash
    curl https://bearlab.ai/v1/chat/completions \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "model": "gpt-4o",
        "messages": [
          {"role": "system", "content": "你是一个简洁的助手。"},
          {"role": "user", "content": "你好"}
        ],
        "temperature": 0.7
      }'
    ```
  </Tab>
  <Tab title="Anthropic 兼容">
    ```bash
    curl https://bearlab.ai/anthropic/v1/messages \
      -H "x-api-key: YOUR_API_KEY" \
      -H "anthropic-version: 2023-06-01" \
      -H "Content-Type: application/json" \
      -d '{
        "model": "claude-sonnet-4-5",
        "max_tokens": 1024,
        "system": "你是一个简洁的助手。",
        "messages": [{"role": "user", "content": "你好"}]
      }'
    ```
  </Tab>
</Tabs>

## 常用参数

| 参数 | 说明 |
| --- | --- |
| `model` | 模型 ID。可用列表见 [模型与定价接口](/zh-Hans/guides/models)。 |
| `messages` | 多轮对话消息数组，按时间顺序排列。 |
| `temperature` | 采样温度，`0`~`2`，越高越发散。 |
| `top_p` | 核采样，与 `temperature` 二选一调节即可。 |
| `max_tokens` | 最大生成 token 数。Anthropic 协议下为**必填**。 |
| `stream` | 是否流式返回，`true` 时以 SSE 推送。 |
| `stop` | 停止序列。 |

<Warning>
  **协议差异**：OpenAI 协议把 system 提示放进 `messages` 的 `role: system`；Anthropic 协议用顶层 `system` 字段，且 `max_tokens` 必填。
</Warning>

## 多轮对话

把历史消息按顺序全部放进 `messages` 即可，服务端不保存上下文，每次请求需自带完整历史。

```json
{
  "model": "gpt-4o",
  "messages": [
    {"role": "user", "content": "我叫小明"},
    {"role": "assistant", "content": "你好，小明！"},
    {"role": "user", "content": "我叫什么？"}
  ]
}
```

## 流式输出

设 `stream: true`，响应以 SSE 分块推送，最后以 `data: [DONE]` 结束（OpenAI 协议）。

```python
from openai import OpenAI

client = OpenAI(api_key="YOUR_API_KEY", base_url="https://bearlab.ai/v1")

stream = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "写一首四行小诗"}],
    stream=True,
)
for chunk in stream:
    delta = chunk.choices[0].delta.content
    if delta:
        print(delta, end="", flush=True)
```

## 响应结构（OpenAI 协议）

```json
{
  "id": "chatcmpl-...",
  "object": "chat.completion",
  "model": "gpt-4o",
  "choices": [
    {
      "index": 0,
      "message": {"role": "assistant", "content": "..."},
      "finish_reason": "stop"
    }
  ],
  "usage": { "prompt_tokens": 23, "completion_tokens": 58, "total_tokens": 81 }
}
```

`usage` 中的 token 数即本次计费依据，详见 [计费与充值](/zh-Hans/guides/pricing)。

## 工具调用（Function Calling）

OpenAI 与 Anthropic 协议都支持工具调用。

<Tabs>
  <Tab title="OpenAI 兼容">
    ```python
    from openai import OpenAI

    client = OpenAI(base_url="https://bearlab.ai/v1", api_key="YOUR_API_KEY")

    tools = [{
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "Get the current weather for a city",
            "parameters": {
                "type": "object",
                "properties": {"city": {"type": "string"}},
                "required": ["city"],
            },
        },
    }]

    resp = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": "What's the weather in London?"}],
        tools=tools,
    )
    print(resp.choices[0].message)
    ```
  </Tab>
  <Tab title="Anthropic 兼容">
    ```python
    import anthropic

    client = anthropic.Anthropic(base_url="https://bearlab.ai/anthropic", api_key="YOUR_API_KEY")

    tools = [{
        "name": "get_weather",
        "description": "Get the current weather for a city",
        "input_schema": {
            "type": "object",
            "properties": {"city": {"type": "string"}},
            "required": ["city"],
        },
    }]

    msg = client.messages.create(
        model="claude-sonnet-4-5",
        max_tokens=1024,
        tools=tools,
        messages=[{"role": "user", "content": "What's the weather in London?"}],
    )
    print(msg.content)
    ```
  </Tab>
</Tabs>

## Gemini 原生协议

Gemini 模型可直接用 Google AI Studio 的原生格式调用，无需转成 OpenAI 格式，适合已在用 `genai` SDK 的项目。

```python
from google import genai
from google.genai import types

client = genai.Client(
    api_key="YOUR_API_KEY",
    http_options=types.HttpOptions(api_version="v1beta", base_url="https://bearlab.ai"),
)

resp = client.models.generate_content(model="gemini-2.5-pro", contents="你好")
print(resp.text)
```

<Warning>
  1. `base_url` 设为 `https://bearlab.ai`（不带 `/v1beta` 后缀，SDK 会自动追加）
  2. `api_version` 设为 `v1beta`
  3. 依赖：Python `pip install google-genai`、Node.js `npm install @google/genai`
</Warning>

<Tip>
  重复的长前缀（固定 system 提示、长文档）可启用 [提示词缓存](/zh-Hans/guides/prompt-caching)，显著降低重复部分的费用。
</Tip>
