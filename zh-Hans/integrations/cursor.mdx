title: "Cursor"

---

[Cursor](https://cursor.com) 是一个流行的 AI 代码编辑器。通过 BearLab AI 的 OpenAI 兼容接口连接。

Cursor 仅支持 OpenAI 协议，因此请始终使用 `/v1` 基础 URL 并使用您的 `sk-...` 密钥进行身份验证。

<Steps>
  <Step title="打开模型设置">
    打开 **Settings → Models**。
  </Step>
  <Step title="覆盖基础 URL">
    在 **OpenAI API Key** 区域中，展开 **Override OpenAI Base URL** 并为您的区域设置值：

    <Tabs>
      <Tab title="US / EU">
        - **Base URL**: `https://bearlab.ai/v1`
        - **API Key**: 您的 `sk-...` 密钥
      </Tab>
      <Tab title="Asia-Pacific">
        - **Base URL**: `https://bearlab.space/v1`
        - **API Key**: 您的 `sk-...` 密钥
      </Tab>
    </Tabs>
  </Step>
  <Step title="添加模型并验证">
    当设置自定义基础 URL 时，Cursor 不会自动拉取模型列表，因此需要手动添加模型名称。使用 [models endpoint](/zh-Hans/guides/models) 中显示的 ID，例如：

    - `gpt-5.5`
    - `claude-sonnet-4-6`
    - `claude-opus-4-7`
    - `gemini-3.1-pro-preview`
    - `deepseek-v4-pro`

    保存，然后点击 **Verify**。
  </Step>
</Steps>

<Note>
  模型名称必须完全匹配 [models endpoint](/zh-Hans/guides/models) 返回的 ID，包括后缀。不存在的名称将无法验证。如果验证仍然失败，请先使用 [Quickstart](/zh-Hans/guides/quickstart) 中的 curl 确认密钥。
</Note>

<Warning>
  选择与您的区域相符的基础 URL。`https://bearlab.ai` 服务美国/欧洲，`https://bearlab.space` 服务亚太地区；路径对两者都是 `/v1`。不要混用一个区域的密钥与另一个区域的基础 URL。
</Warning>
