[Codex CLI](https://github.com/openai/codex) 是 OpenAI 官方的命令行代码代理。您可以通过添加指向 BearLab AI 网关的自定义提供者来将其连接到 BearLab AI。

<Warning>
  Codex 使用 OpenAI **Responses API**（`wire_api = "responses"`），这是 OpenAI 专有协议，**仅适用于 OpenAI 系列模型**（`gpt-*` / `gpt-oss-*`）。Claude、Gemini、GLM 及其他非 OpenAI 模型无法通过 Codex 驱动 — 请改用 [Claude Code](/zh-Hans/guides/claude-code)、[Cursor](/zh-Hans/guides/cursor) 或 [Cline](/zh-Hans/guides/cline)。
</Warning>

## 基础 URL

Codex 与 OpenAI 兼容端点通信，因此 `base_url` 以 `/v1` 结尾。选择最接近您的地区：

| 地区 | `base_url` |
| --- | --- |
| 美国 / 欧洲 | `https://bearlab.ai/v1` |
| 亚太地区 | `https://bearlab.space/v1` |

## 推荐模型

使用 OpenAI 系列的模型 ID，必须完全符合您的[模型列表](/zh-Hans/guides/models)中的显示方式。可用的 OpenAI 系列 ID 包括：

| 模型 | ID | 最适用于 |
| --- | --- | --- |
| GPT-5.5 | `gpt-5.5` | 最新通用模型，Codex 的强大默认选择 |
| GPT-5.4 | `gpt-5.4` | 品质与速度均衡 |
| GPT-5.4-mini | `gpt-5.4-mini` | 轻量且快速，适合子任务与交互编辑 |
| GPT-5.2 | `gpt-5.2` | 可靠的日常变更工作马 |
| GPT-5 | `gpt-5` | 稳定的基准，兼容性广泛 |

<Note>
  模型 ID 区分大小写，必须完全符合[模型端点](/zh-Hans/guides/models)。配置 Codex 前，请先执行快速列表以确认您的密钥可以看到哪些模型。
</Note>

## 配置

编辑 `~/.codex/config.toml` 并添加指向 BearLab AI 的提供者。

<Tabs>
  <Tab title="美国 / 欧洲">
    ```toml
    model = "gpt-5.5"
    model_provider = "bearlab"

    # Third-party providers must disable response storage
    disable_response_storage = true

    [model_providers.bearlab]
    name = "BearLab AI"
    base_url = "https://bearlab.ai/v1"
    env_key = "BEARLAB_API_KEY"
    wire_api = "responses"
    ```
  </Tab>
  <Tab title="亚太地区">
    ```toml
    model = "gpt-5.5"
    model_provider = "bearlab"

    # Third-party providers must disable response storage
    disable_response_storage = true

    [model_providers.bearlab]
    name = "BearLab AI"
    base_url = "https://bearlab.space/v1"
    env_key = "BEARLAB_API_KEY"
    wire_api = "responses"
    ```
  </Tab>
</Tabs>

导出您的 API 密钥并运行：

```bash
export BEARLAB_API_KEY="sk-..."
codex
```

<Note>
  使用以 `/v1` 结尾的 OpenAI 兼容地址。第三方 OpenAI 格式提供者必须配置 `wire_api = "responses"` 和 `disable_response_storage = true`。认证是 Bearer 密钥（`sk-...`），由 Codex 通过 `env_key` 在 `Authorization` 标头中发送。`model` 必须是您[模型列表](/zh-Hans/guides/models)中存在的 OpenAI 系列 ID。
</Note>
