## 请求头

| 请求头 | 必填 | 说明 |
| --- | --- | --- |
| `Authorization` | ✓ | `Bearer YOUR_ACCESS_TOKEN`，账号访问令牌（控制台登录态，非 `sk-` API Key） |

接口：`GET /api/user/self/groups`

列出当前用户可用的分组及其计费倍率。返回对象的**键**即分组名，创建 / 更新令牌时作为 `group` 字段传入。

```bash
curl "https://bearlab.ai/api/user/self/groups" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```json Response
{
  "success": true,
  "message": "",
  "data": {
    "default": { "ratio": 1, "desc": "默认分组" },
    "vip": { "ratio": 0.8, "desc": "VIP 分组" },
    "auto": { "ratio": "自动", "desc": "自动选择可用分组" }
  }
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `data` | object | 键为分组名，值为分组信息 `{ ratio, desc }` |
| `data.<group>.ratio` | number \| string | 该分组计费倍率；`auto` 分组为字符串 `"自动"` |
| `data.<group>.desc` | string | 分组描述 |

<Note>
来源：new-api 源码 `controller/group.go` `GetUserGroups`（路由 `GET /api/user/self/groups`，需登录鉴权）。响应 `data` 是 `{ 分组名: { ratio, desc } }` 映射，非数组。旧文档的 `GET /api/token/group` 路径在 new-api 中**不存在**。
</Note>
