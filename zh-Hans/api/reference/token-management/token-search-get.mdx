---
title: "搜索 Token"
description: "GET /api/token/search"
---

## 请求头

| 请求头 | 必填 | 说明 |
| --- | --- | --- |
| `Authorization` | ✓ | `Bearer YOUR_ACCESS_TOKEN`，账号访问令牌（控制台登录态，非 `sk-` API Key） |

接口：`GET /api/token/search`

按名称或 Key 前缀搜索当前账号的令牌，结果分页返回。

## 查询参数

| 查询参数 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| `keyword` |  | string | 按 Token 名称模糊搜索 |
| `token` |  | string | 按 Key 前缀筛选 |
| `p` |  | integer | 页码，从 1 开始 |
| `page_size` |  | integer | 每页数量 |

```bash
curl "https://bearlab.ai/api/token/search?keyword=my-token" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```json 回应
{
  "success": true,
  "message": "",
  "data": {
    "items": [
      {
        "id": 10623,
        "user_id": 42,
        "key": "ntEH**********7nAh",
        "status": 1,
        "name": "my-token",
        "created_time": 1781977779,
        "expired_time": -1,
        "remain_quota": 0,
        "unlimited_quota": true,
        "used_quota": 0,
        "group": ""
      }
    ],
    "page": 1,
    "page_size": 10,
    "total": 1

}
```

响应结构与 `GET /api/token/` 相同（`key` 已脱敏，字段说明见 [Token 列表](/api/reference/token-management/token-get)）。

<Note>
来源：new-api 源码 `controller/token.go` `SearchTokens`。接受 `keyword`（名称）与 `token`（Key 前缀）两个搜索参数加分页。注意：`GET /api/token/` 列表接口**不**接受 `keyword`，模糊搜索须用本接口。
</Note>
