---
title: "获取 Token 详情"
description: "GET /api/token/{id}"
---

## 请求头

| 请求头 | 必填 | 说明 |
| --- | --- | --- |
| `Authorization` | ✓ | `Bearer YOUR_ACCESS_TOKEN`，账号访问令牌（控制台登录态，非 `sk-` API Key） |

接口：`GET /api/token/{id}`

根据 Token ID 获取单个令牌的配置详情。响应中的 `key` 已脱敏；获取**完整明文 Key** 请用 `POST /api/token/{id}/key`。

## 路径参数

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `id` | ✓ | Token ID，来自 `GET /api/token/` 的 `data.items[].id` |

```bash
curl https://bearlab.ai/api/token/10665 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```json 回应
{
  "success": true,
  "message": "",
  "data": {
    "id": 10665,
    "user_id": 42,
    "key": "VrqY**********tE4I",
    "status": 1,
    "name": "my-token",
    "created_time": 1781977779,
    "accessed_time": 1781977779,
    "expired_time": -1,
    "remain_quota": 0,
    "unlimited_quota": true,
    "used_quota": 0,
    "model_limits_enabled": false,
    "model_limits": "",
    "allow_ips": "",
    "group": "free-play",
    "cross_group_retry": false
  }
}
```

响应为脱敏后的完整 Token 对象，字段含义见 [Token 列表](/api/reference/token-management/token-get)。

<Note>
来源：new-api 源码 `controller/token.go` `GetToken`，响应 `data` 为 `buildMaskedTokenResponse(token)`（`key` 已脱敏）。获取完整明文 Key 是独立接口 `POST /api/token/{id}/key`（`GetTokenKey`）。
</Note>
