title: "個人ログの取得"
description: "GET /api/log/self"
---

## リクエストヘッダー

| リクエストヘッダー | 必須 | 説明 |
| --- | --- | --- |
| `Authorization` | ✓ | `Bearer YOUR_ACCESS_TOKEN`、アカウントアクセストークン（コンソールログイン状態、`sk-` API Key ではありません） |

エンドポイント：`GET /api/log/self`

現在のアカウントのAPI呼び出しログをページネーション付きで検索します。

## クエリパラメータ

| クエリパラメータ | 必須 | 型 | 説明 |
| --- | --- | --- | --- |
| `p` |  | integer | ページ番号、1 から開始 |
| `page_size` |  | integer | 1ページあたりの件数 |
| `type` |  | integer | ログタイプ（`2` 消費 / `1` チャージ / `3` 管理 / `4` システム / `5` エラー；省略または `0` はすべて） |
| `token_name` |  | string | トークン名でフィルター |
| `model_name` |  | string | モデル名でフィルター |
| `group` |  | string | グループでフィルター |
| `start_timestamp` |  | integer | 開始時刻、Unix 秒 |
| `end_timestamp` |  | integer | 終了時刻、Unix 秒 |
| `request_id` |  | string | リクエスト ID で正確に検索 |
| `upstream_request_id` |  | string | アップストリームリクエスト ID で検索 |

```bash
curl "https://bearlab.ai/api/log/self?p=1&page_size=10&type=2" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `data.items` | array | ログレコード配列 |
| `type` | integer | ログタイプ（上記と同じ） |
| `content` | string | ログ説明（計費倍率など） |
| `token_name` | string | トークン名 |
| `model_name` | string | モデル名 |
| `quota` | integer | 今回の消費額度 |
| `prompt_tokens` / `completion_tokens` | integer | 入力 / 出力 トークン数 |
| `use_time` | integer | リクエスト実行時間（秒） |
| `is_stream` | boolean | ストリーミングかどうか |
| `channel` | integer | チャネル ID（注意：JSON フィールド名は `channel`、`channel_id` ではありません） |
| `group` | string | 使用されたグループ |
| `request_id` | string | リクエスト ID |
| `data.page` / `page_size` / `total` | integer | ページネーション情報 |

<Note>
ソース：new-api ソースコード `controller/log.go` の `GetUserLogs` + `model/log.go` の `Log`。個人ログ検索は `keyword` または `channel` クエリパラメータをサポート**していません**（`/api/log/self/search` は廃止されました）；`channel` は応答フィールドとしてのみ返されます。
</Note>
