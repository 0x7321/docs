---
title: "レート制限"
description: "制限の形態、ユーザーレベルとの関係、対応方法。"
---

リソース使用を公平に保ち、サービスを安定に保つために、プラットフォームはリクエスト頻度とトークン使用を制限しています。

## 制限の形態

| 形態 | 意味 |
| --- | --- |
| **RPM** | 1分あたりの最大リクエスト |
| **RPH** | 1時間あたりの最大リクエスト |
| **RPD** | 1日あたりの最大リクエスト |
| **TPM** | 1分あたりの最大トークン(入力+出力) |
| **TPD** | 1日あたりの最大トークン |

## ユーザーレベルとの関係

制限は主に[ユーザーレベル](/ja/guides/user-levels)に左右されます。

| 層 | 取得方法 | 制限 |
| --- | --- | --- |
| **Tier 1** | 登録後のデフォルト | 5 RPM |
| **Tier 2** | いずれかの入金後に自動 | 制限なし |

新規アカウントはTier 1(5 RPM)でデフォルト。**いずれかの入金でTier 2に即座にアップグレードされ、レート制限が削除されます**。手動リクエストは不要です。

## TPMとmax_tokens

リクエスト送信時には合計トークン数を知ることができないため、システムはTPM制限のための`max_tokens`パラメータで推定し、生成後に実際のトークンで修正します。`max_tokens`を**実際の必要に近く**設定して、過度推定からのエラーを減らします。

## 制限に達したとき

リクエストまたはトークン使用が上限に達すると、APIは`429 Too Many Requests`を返します。後のリクエストはクールダウンが経過するまで一時的に拒否されます。

```http
HTTP/1.1 429 Too Many Requests
Content-Type: application/json

{ "error": { "message": "The current group upstream load is saturated, please try again later", "type": "bearlab_api_error" } }
```

## ヒント

- **指数バックオフ**: `429`で、待機して倍のディレイで再試行します。
- **バッチリクエスト**: リクエストが制限されているがトークンが制限されていない場合、タスクを1つのリクエストにマージします。
- **詳細が必要**: アカウントマネージャーに連絡して専用クォータを取得します。
