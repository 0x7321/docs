モデルを呼び出すには 3 つのステップがあります。キーを作成し、クライアントを正しいベース URL に指定してから、最初のチャットリクエストを送信します。

<Note>
  地域に応じてベース URL を選択してください。**米国 / EU：** `https://bearlab.ai`。**アジア太平洋：** `https://bearlab.space`。その他すべて（キー、モデル、リクエスト形式）は地域全体で同じです。[サービスの可用性](/ja/guides/availability)を参照してください。
</Note>

<Steps>
  <Step title="サインアップとトップアップ">
    [コンソール](https://bearlab.ai)で登録し、ウォレットページで資金を追加してください。新規ユーザーは少量の入金でテストを開始できます。
  </Step>
  <Step title="API キーを作成">
    ダッシュボード、API トークンに移動し、トークンを追加をクリックして、`sk-...` キーをコピーしてください。キーは最初の 1 回だけ完全に表示されます。毎回のリクエストで `Authorization: Bearer sk-...` として渡します。
  </Step>
  <Step title="ベース URL を設定">
    あなたの地域とプロトコルに応じたベース URL を使用してください：

    | プロトコル | 米国 / EU | アジア太平洋 |
    | --- | --- | --- |
    | OpenAI 互換 | `https://bearlab.ai/v1` | `https://bearlab.space/v1` |
    | Anthropic | `https://bearlab.ai/v1/messages` | `https://bearlab.space/v1/messages` |
  </Step>
  <Step title="最初のリクエストを実行">
    下のサンプルを送信してください。`sk-...` を作成したばかりのキーに置き換えてください。
  </Step>
</Steps>

## 例

OpenAI 互換エンドポイントは `POST /v1/chat/completions`；Anthropic エンドポイントは `POST /v1/messages`。どちらも `Authorization: Bearer sk-...` で認証します。`model` 値は、あなたのキーに対して[モデルリスト](/ja/guides/models)が返すモデルの 1 つである必要があります。以下の例では、このアカウントで利用可能なモデルを使用しています。

### OpenAI 互換

<CodeGroup>
```bash US / EU (bearlab.ai)
curl https://bearlab.ai/v1/chat/completions \
  -H "Authorization: Bearer sk-..." \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.4",
    "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
  }'
```

```bash Asia-Pacific (bearlab.space)
curl https://bearlab.space/v1/chat/completions \
  -H "Authorization: Bearer sk-..." \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.4",
    "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
  }'
```
</CodeGroup>

### Anthropic

Anthropic エンドポイントは `max_tokens` と `anthropic-version` ヘッダーを期待します。`Authorization: Bearer sk-...` で認証できます（ゲートウェイは `x-api-key: sk-...` も受け入れます）。

<CodeGroup>
```bash US / EU (bearlab.ai)
curl https://bearlab.ai/v1/messages \
  -H "Authorization: Bearer sk-..." \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4-6",
    "max_tokens": 256,
    "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
  }'
```

```bash Asia-Pacific (bearlab.space)
curl https://bearlab.space/v1/messages \
  -H "Authorization: Bearer sk-..." \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4-6",
    "max_tokens": 256,
    "messages": [{"role": "user", "content": "Introduce yourself in one sentence"}]
  }'
```
</CodeGroup>

### Python（OpenAI SDK）

地域の `/v1` ベース URL にまで SDK をポイントしてください。SDK は `/chat/completions` を自動的に追加します。

<CodeGroup>
```python US / EU (bearlab.ai)
from openai import OpenAI

client = OpenAI(
    api_key="sk-...",
    base_url="https://bearlab.ai/v1",
)

resp = client.chat.completions.create(
    model="gpt-5.4",
    messages=[{"role": "user", "content": "Introduce yourself in one sentence"}],
)
print(resp.choices[0].message.content)
```

```python Asia-Pacific (bearlab.space)
from openai import OpenAI

client = OpenAI(
    api_key="sk-...",
    base_url="https://bearlab.space/v1",
)

resp = client.chat.completions.create(
    model="gpt-5.4",
    messages=[{"role": "user", "content": "Introduce yourself in one sentence"}],
)
print(resp.choices[0].message.content)
```
</CodeGroup>
