各モデルは**複数のプロバイダー**によってサポートされている場合があります。サービスノードはソースとコストで整理され、異なる価格倍数とモデルカバレッジを持つものです。システムはトークン設定に基づいて利用可能なプロバイダーを選択します。1つが失敗またはタイムアウトした場合、別のプロバイダーに自動的に再試行され、コードに対しては透過的です。

## 利用可能なプロバイダー

| ID | 名前 | 説明 |
| --- | --- | --- |
| `official` | 公式 | 主要な公式プラットフォーム(OpenAI、Anthropic、Google)への直接接続をバンドルするプロバイダーグループ; 最高の安定性 |
| `official_cheap` | 公式(割引) | コスト最適化された公式チャネル; より低い価格で高い安定性 |
| `mix` | 混合 | マルチソース混合チャネル; 最低価格、最広カバレッジ |

`official`は主要な公式プラットフォームのノードをバンドルする**プロバイダーグループ**です。これを選択すると、各ベンダーを指定することなく、すべての公式ノード全体でルーティングとフェイルオーバーが行われます。

### 選択方法

| 必要な場合 | 推奨設定 |
| --- | --- |
| 最大の安定性、価格は関係ない | `official` |
| 良い安定性、より良い価値 | `official_cheap` |
| 最低コスト、わずかな不安定性は許容できる | `mix` |
| 割引公式優先、混合にフォールバック | `official_cheap,mix` |

## トークンで設定

トークンを作成または編集するときに、デフォルトのルーティングを設定します。

- **プロバイダーリスト**: 空(推奨)はすべての利用可能なプロバイダー全体でルーティング; 1つのみはそれに制限; 複数(カンマ区切り)は順序またはソート戦略で試みられます。
- **プロバイダー専用モード**: オフ(デフォルト)は指定されたプロバイダーを優先するがフォールバック; オンはそれらに厳密に制限し、すべてが失敗した場合はエラーを返します。
- **ソート戦略**: 手動(デフォルト)、遅延、価格、またはスループット。

## リクエストごとにオーバーライド

ヘッダー経由でトークン設定を単一のリクエストについてオーバーライドします。

| ヘッダー | 効果 |
| --- | --- |
| `X-Provider-Order` | このリクエストのプロバイダーと順序 |
| `X-Provider-Only` | `"true"`は厳密モード、フォールバックなし |
| `X-Provider-Sort` | ソート戦略をオーバーライド(`latency` / `price` / `throughput`) |
| `X-Provider-Ignore` | スキップするプロバイダー、トークン設定に積み重ねられます |

> ヘッダーで指定されたプロバイダーは、アカウントの権限の対象となります。

### 例

```bash
curl https://bearlab.ai/v1/chat/completions \
  -H "Authorization: Bearer sk-your-token" \
  -H "X-Provider-Order: official_cheap,mix" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o", "messages": [{"role": "user", "content": "Hello"}]}'
```
