---
title: "エラーハンドリング"
description: "一般的なHTTPステータスコード、その意味、トラブルシューティング方法。"
---

APIは標準的なHTTPステータスコードを使用します。エラー本文はJSONであり、リクエストIDヘッダー([キャッシュルーティング](/ja/guides/prompt-caching)のためのトラブルシューティング)を含みます。

```json
{
  "error": {
    "code": "",
    "message": "A description, including the request id",
    "type": "bearlab_api_error"
  }
}
```

## 一般的なステータスコード

| コード | 意味 | 原因と修正 |
| --- | --- | --- |
| `400` | 不正なパラメータ | 不正なまたは間違ったタイプのパラメータ; Anthropicで`max_tokens`が不正です。`message`に従い修正してください。 |
| `401` | 認証失敗 | APIキーが正しく設定されていません(間違った/削除されたキー、またはBearerとX-api-keyの不一致)。 |
| `403` | 残高不足またはアクセス権限なし | 残高不足; またはモデル/グループへのアクセスがありません。最も一般的な権限の問題は、本人確認が必要なモデルです。 |
| `429` | レート制限 | RPM / RPD / TPM / TPDに達しました。[レート制限](/ja/guides/rate-limits)を参照してください。不正なパラメータからの繰り返される再試行の場合もあります。 |
| `503` / `504` | 高負荷 | 後で再試行してください。チャットの場合は、`"stream": true`でストリーミングを試みてください。 |
| `500` | 不明なサーバーエラー | リクエストIDを記載してサポートにお問い合わせください。 |

## チェックリスト

<Steps>
  <Step title="error.messageを最初に読む">
    ほとんどの問題はそこで説明されています。
  </Step>
  <Step title="base_urlとプロトコルを確認">
    `/v1` (OpenAI) vs `/v1/messages` (Anthropic); 認証ヘッダーが一致する必要があります。
  </Step>
  <Step title="モデル名を確認">
    [modelsエンドポイント](/ja/guides/models)経由で`model`が存在することを確認してください。
  </Step>
  <Step title="残高を確認 / 再試行">
    `403`は残高の場合があります。`5xx` / `429`を指数バックオフで再試行してから、サポートに連絡してください。
  </Step>
</Steps>

<Note>
  `stream: true`の場合、エラーはHTTPステータスではなく、接続が開いた後SSEイベントとして到達する場合があります。ストリーム中盤の`error`イベントを処理してください。
</Note>
