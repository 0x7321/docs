---

[Codex CLI](https://github.com/openai/codex) は OpenAI 公式のコマンドラインコード エージェントです。BearLab AI ゲートウェイを指す カスタムプロバイダを追加することで、BearLab AI に接続できます。

<Warning>
  Codex は OpenAI **Responses API**（`wire_api = "responses"`）を使用します。これは OpenAI 専有プロトコルで、**OpenAI ファミリーモデルのみで機能します**（`gpt-*` / `gpt-oss-*`）。Claude、Gemini、GLM および その他の非 OpenAI モデルは Codex で駆動できません — 代わりに [Claude Code](/ja/guides/claude-code)、[Cursor](/ja/guides/cursor)、または [Cline](/ja/guides/cline) を使用してください。
</Warning>

## ベース URL

Codex は OpenAI 互換エンドポイントと通信するため、`base_url` は `/v1` で終わります。あなたに最も近いリージョンを選択してください：

| リージョン | `base_url` |
| --- | --- |
| 米国 / ヨーロッパ | `https://bearlab.ai/v1` |
| アジア太平洋 | `https://bearlab.space/v1` |

## 推奨モデル

OpenAI ファミリーのモデル ID を使用してください。[モデルリスト](/ja/guides/models)に表示されているとおりに完全に一致する必要があります。利用可能な OpenAI ファミリー ID は以下の通りです：

| モデル | ID | 最適な用途 |
| --- | --- | --- |
| GPT-5.5 | `gpt-5.5` | 最新の汎用モデル、Codex の強力なデフォルト |
| GPT-5.4 | `gpt-5.4` | 品質とスピードのバランス |
| GPT-5.4-mini | `gpt-5.4-mini` | 軽量で高速、サブタスクと対話的編集用 |
| GPT-5.2 | `gpt-5.2` | 日常的な変更に信頼できるワークホース |
| GPT-5 | `gpt-5` | 安定したベースライン、幅広い互換性 |

<Note>
  モデル ID は大文字と小文字を区別し、[モデル エンドポイント](/ja/guides/models)と完全に一致する必要があります。Codex を設定する前に、クイックリストを実行してあなたのキーで何が見えるかを確認してください。
</Note>

## 設定

`~/.codex/config.toml` を編集し、BearLab AI を指すプロバイダを追加してください。

<Tabs>
  <Tab title="米国 / ヨーロッパ">
    ```toml
    model = "gpt-5.5"
    model_provider = "bearlab"

    # Third-party providers must disable response storage
    disable_response_storage = true

    [model_providers.bearlab]
    name = "BearLab AI"
    base_url = "https://bearlab.ai/v1"
    env_key = "BEARLAB_API_KEY"
    wire_api = "responses"
    ```
  </Tab>
  <Tab title="アジア太平洋">
    ```toml
    model = "gpt-5.5"
    model_provider = "bearlab"

    # Third-party providers must disable response storage
    disable_response_storage = true

    [model_providers.bearlab]
    name = "BearLab AI"
    base_url = "https://bearlab.space/v1"
    env_key = "BEARLAB_API_KEY"
    wire_api = "responses"
    ```
  </Tab>
</Tabs>

あなたの API キーをエクスポートして実行してください：

```bash
export BEARLAB_API_KEY="sk-..."
codex
```

<Note>
  `/v1` で終わる OpenAI 互換アドレスを使用してください。サードパーティ OpenAI 形式プロバイダの場合、`wire_api = "responses"` と `disable_response_storage = true` が必須です。認証は Bearer キー（`sk-...`）で、Codex が `env_key` 経由で `Authorization` ヘッダで送信します。`model` は [モデルリスト](/ja/guides/models)に存在する OpenAI ファミリー ID である必要があります。
</Note>
