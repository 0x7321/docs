---
title: "错误处理"
description: "常见 HTTP 状态码、含义与排查方法。"
---

接口遵循标准 HTTP 状态码，错误响应体为 JSON，并带一个请求 ID 头（用于排查与[缓存路由](/guides/prompt-caching)），通常形如：

```json
{
  "error": {
    "code": "",
    "message": "具体错误描述（含 request id）",
    "type": "bearlab_api_error"
  }
}
```

## 常见状态码

| 状态码 | 含义 | 常见原因与排查 |
| --- | --- | --- |
| `400` | 参数错误 | 参数缺失 / 类型错误；Anthropic 协议漏填 `max_tokens`。按 `message` 修正。 |
| `401` | 鉴权失败 | API Key 未正确设置（密钥错误 / 已删除，或 `Bearer` 与 `x-api-key` 混用）。 |
| `403` | 余额不足或无权限 | 余额耗尽；或无权访问该模型 / 分组。最常见的无权限是**该模型需要实名认证**。 |
| `429` | 触发限速 | RPM / RPD / TPM / TPD 等达上限，见 [速率限制](/guides/rate-limits)；也可能是参数错误导致反复重试触发。 |
| `503` / `504` | 系统负载高 | 稍后重试；对话请求可改用流式 `"stream": true`。 |
| `500` | 服务未知错误 | 带上 request id 联系客服排查。 |

## 排查清单

<Steps>
  <Step title="先看 error.message">
    大多数问题描述里已写明原因。
  </Step>
  <Step title="核对 base_url 与协议">
    `/v1`（OpenAI）还是 `/anthropic`（Anthropic），鉴权头要匹配。
  </Step>
  <Step title="核对模型名">
    用 [模型接口](/guides/models) 确认 `model` 拼写存在。
  </Step>
  <Step title="检查余额 / 重试">
    `402` 即余额问题；`5xx` / `429` 用指数退避重试，持续失败再联系客服。
  </Step>
</Steps>

<Note>
  `stream: true` 时，错误可能在建立连接后以 SSE 事件返回。客户端应处理流中途出现的 `error` 事件，而不仅依赖 HTTP 状态码。
</Note>
