## 什麼是提示詞快取

很多模型供應商（包括 Anthropic Claude 和 OpenAI）原生支援提示詞快取。當同一段上下文被重複傳送時，供應商會複用此前的計算結果，**降低延遲和 token 成本**。

## 快取的限制

**快取不能跨賬戶共享**，這是供應商側的限制，不是平臺限制。

作為多渠道中轉平臺，我們無法保證同一賬戶連續兩次請求一定路由到同一後端渠道。若被髮到不同渠道，已有快取無法複用，命中率下降、成本上升。

## 保持路由一致

為此，你可以在後續請求裡帶上前一次請求的 ID，系統會盡量把新請求路由到同一渠道，最大化快取複用。兩種方式：

| 請求頭 | 取值來源 | 有效期 |
| --- | --- | --- |
| `X-Request-Id` | 上一次響應返回的請求 ID 頭 | 5 分鐘 |
| `X-Response-Id` | 上一次響應 JSON 裡的會話 ID（如 Claude 的 `message.id`） | 5 分鐘 |

### 示例

```python
import anthropic

client = anthropic.Anthropic(
    base_url="https://bearlab.ai/anthropic",
    api_key="API_KEY",
)

# 第一次請求
response = client.messages.create(
    model="claude-sonnet-4-5",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello!"}],
)
response_id = response.id

# 第二次請求 —— 帶上 response id 以留在同一渠道
response2 = client.messages.create(
    model="claude-sonnet-4-5",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "Hello!"},
        {"role": "assistant", "content": response.content[0].text},
        {"role": "user", "content": "Continue our conversation..."},
    ],
    extra_headers={"X-Response-Id": response_id},
)
```

## 使用建議

把**不變內容放前面**（system、長文件），變化內容放後面（使用者最新提問），以最大化字首命中。字首越長、複用越頻繁，省得越多；一次性請求無需快取。
