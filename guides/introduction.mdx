> BearLab AI 將來自數十家領先提供商的對話模型整合在一個 API 下。OpenAI、Anthropic、Google、DeepSeek 等都可以用單一密鑰存取。

## 什麼是 BearLab AI

BearLab AI 是一個對話模型閘道。您不需要分別向 OpenAI、Anthropic、Google 等註冊、充值和管理密鑰，只需在這裡建立一個密鑰，指向一個 `base_url`，即可呼叫它們最新的對話模型。

該閘道相容於 OpenAI 和 Anthropic 協定，因此現有代碼通常只需更改端點和密鑰即可使用——無須進行其他修改。

我們專注於對話模型這一項功能，這使得介面簡潔、計費清晰，服務也更加穩定。

## 基礎 URL

選擇離您最近的地區。每個地區都在兩種協定後面公開相同的模型。

<Tabs>
  <Tab title="US / EU">
    | 協定 | 基礎 URL |
    | --- | --- |
    | OpenAI 相容 | `https://bearlab.ai/v1` |
    | Anthropic | `https://bearlab.ai/v1/messages` |
  </Tab>
  <Tab title="Asia-Pacific">
    | 協定 | 基礎 URL |
    | --- | --- |
    | OpenAI 相容 | `https://bearlab.space/v1` |
    | Anthropic | `https://bearlab.space/v1/messages` |
  </Tab>
</Tabs>

<Note>
這兩個地區可以互換——同一個密鑰在兩個地區都可用。按網路延遲選擇，而非功能，因為模型目錄和協定完全相同。
</Note>

## 驗證

每個請求都在 `Authorization` 標頭中以 Bearer token 形式攜帶您的 API 密鑰。密鑰以 `sk-` 開頭。

<CodeGroup>

```bash OpenAI-compatible
curl https://bearlab.ai/v1/chat/completions \
  -H "Authorization: Bearer sk-..." \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.5",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

```bash Anthropic
curl https://bearlab.ai/v1/messages \
  -H "Authorization: Bearer sk-..." \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-opus-4-8",
    "max_tokens": 1024,
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

```bash Asia-Pacific
curl https://bearlab.space/v1/chat/completions \
  -H "Authorization: Bearer sk-..." \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.5",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

</CodeGroup>

Anthropic 端點另外需要 `anthropic-version` 標頭，需與上游 Anthropic Messages API 相符。

## 主要功能

**一個密鑰支援所有模型**
透過統一介面存取多家廠商和模型。一個帳戶管理所有存取、一個帳戶管理所有支出、一個地方檢視所有使用情況。

**OpenAI / Anthropic 雙協定支援**
兩種協定都原生支援。Cursor、Claude Code、Cline 和 Codex 只需設定基礎 URL 和密鑰即可連線——無須在意模型後面是哪個廠商。暴露 Anthropic 協定的模型同時標記為 `openai` 和 `anthropic` 端點；其他模型僅支援 OpenAI。

**廣泛的模型覆蓋**
模型目錄涵蓋 GPT（5.x 系列）、Claude（Opus / Sonnet / Haiku）、Gemini（3.x）、Grok、DeepSeek、GLM、Qwen、Kimi 和其他對話模型。您能看到的確切列表取決於您密鑰所屬的組別。

**穩定且高可用**
熱門模型由多個上游通道支援。如果其中一個失敗，請求會自動轉移到另一個。

**按使用量付費**
只為使用的部分付費。無月費，無最低充值要求。價格與上游提供商同步，餘額實時可見。

**可觀測的使用情況**
每次呼叫都會記錄其請求、回應和令牌消耗，便於故障排除和成本核算。

## 僅支援對話模型

BearLab AI 專注於對話模型，不定位為通用的圖像、音訊、視訊、音樂或嵌入平台。目標是讓對話模型保持穩定和完整。對於其他功能，請使用專門的平台。
